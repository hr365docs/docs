    • Custom level Permission
      To set up custom-level permissions, follow these steps:
  • Open the site where the application is deployed and click on the settings icon located in the top right corner.


  • Select "Site permissions" and then click on "Advanced permissions settings."


  • Choose "Permissions Levels" and click on "Add a permission level."


  • Click on “Add a Permission Level” and provide a name for the custom permissions.


  • Select "Add items," "Edit items," "Delete items," and "View items." Then, scroll down and click on "Create."


  • Return to the previous page and create a group.

  • Enter the group's name and select the custom permission created in step 3. Click "create."


  • Once the group is created, add users to this group.

  • Click on "New," and enter the names of the users who will access the application. You can also select "Everyone except external users" to grant access to all users automatically, reducing manual additions and you can also add any SharePoint groups directly if needed.


  • Use the "Check permissions" feature to verify the granted permissions, ensuring only the custom permission created in step 4 is assigned.

  • Navigate to the pages, click on the "Share" button, and enter the name of the group created in step 5. Ensure that only view access is granted to the group.

    • These steps will help you grant custom-level permissions efficiently.

    How to Download and Share a HAR File for Support?

    When you experience issues such as being unable to add users for the KPI cycle or appraisal cycle, or if entered data disappears after refreshing the page, the support team may request a HAR file. A HAR (HTTP Archive) file captures detailed logs of your browser’s interactions with the system. This helps us analyse what happened on your screen and identify the root cause quickly and accurately.

    Why the Support Team Needs a HAR File

    A HAR file allows us to:

    1. Review each action taken in your browser.
    2. Detect any network-related issues, failures, or delays.
    3. Identify technical errors that may not be visible on the screen
    How to Download a HAR File

    The steps to open Developer Tools and save a HAR file differ based on your operating system.

    For Windows Users:
    1. Open Google Chrome or Microsoft Edge.
    2. Navigate to the PM365 page where the issue occurs.
    3. Open Developer Tools using any one of the following:
      * Press F12.
      * Press Ctrl + Shift + I
      * Click the three dots menu (⋮) → More Tools → Developer Tools.
    4. Click on the Network tab inside the Developer Tools panel.
    5. Ensure that you clear the logs in network tab (as shown in the clips).
    6. Reproduce the same scenario where you are encountering the issue.
    7. Click on download and download the HAR file.

    For Mac Users:
    1. Open Google Chrome or Microsoft Edge.
    2. Navigate to the PM365 page where the issue occurs.
    3. Open Developer Tools using one of the following methods:
      • Press Command + Option + I
      • Click the three dots menu (⋮) → More Tools → Developer Tools
    4. Click on the Network tab inside the Developer Tools panel.
    5. Ensure that you clear the logs in the Network tab (as shown in the clips).
    6. Reproduce the same scenario where you are encountering the issue.
    7. Click on Download and download the HAR file.

    How can I downgrade to a previous version of the Performance Management 365 application?

    Follow these steps to safely revert to an earlier version of the Performance Management 365 application:

    • Access the AppCatalog directly via the following link:
      https://yourtenantname.sharepoint.com/sites/appcatalog/_layouts/15/tenantAppCatalog.aspx/manageApps

    • Navigate to the AppCatalog and search for the Performance Management 365 application.


    • Select the application and click on More Options.


    • Click on Version History.


    • Select the version you wish to restore, then choose the appropriate time and date.


    • Options such as View, Restore, and Delete will be displayed.


    • Click Restore and confirm the action by selecting OK.


    • Once restored, verify the version number displayed below the selected time and date.


    • After confirmation, select the package, click on Files, and choose Deploy.


    • In the deployment popup, check the required checkbox and click Deploy to finalize the process.
