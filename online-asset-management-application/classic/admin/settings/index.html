Users


Admin can add user to this application. There are different types of roles like, user, admin, asset manager, supervisor and no access as shown in below image.



  • User: Only able to view assigned assets to them.
  • Power user: Able to see all the assets only.
  • Asset Manager: Able to add assets, delete assets, assign assets and return assets, audit assets,report assets, book assets, edit or delete book assets and depreciation.
  • Supervisor: Able to assign assets, return assets, audit assets, report assets book assets, edit or delete book assets and depreciation.
  • No Access: User cannot access the application.

  1. Add
    To add users, click on users under settings from side panel, page will open, click on add tab, add users page will open, First select the user type (M365 users or Non M365 users). For M365 users details fill fetch from M365 automatically if exists, you can edit the details further and click on submit button. For non M365 users, need to add all details manually.
    Note: Assign & return assets notification for non M365 users works on Power Automate setup


  2. Bulk Add
    Download the sample file and fill the email Ids and role as per specified format in the sample and save it. Choose the same file and click on save button
    Note: Add only email(s) and role for M365 users, no need to add details if user’s details are available in O365. If user is M365 user, update “No” in IsNonM365User column. You can add both M365 and non M365 users from bulk add.


  3. Edit User Role
    We can edit user role for either individual or multiple users as shown in below image. Select multiple users to assign same role.
  4. Delete User
    Select the users and click on delete users.
  5. Sync from M365
    If you click on sync from M365 tab, the user's detail in table will update from O365

    1. Currency: From here, you can select the type of currency from the dropdown. Click on the Save button.


    2. Date Format: From here, you can select the date format from the dropdown. Click on the Save button.


    3. Asset ID Prefix: Asset ID Prefix specifies the letter prefix for the generated Asset ID.


    4. First Asset ID: The asset ID is the unique number that will start after you have entered it. Make sure that you must enter the number with at least 3 digits.


    5. Maintenance Module Title Text:


    6. Depreciation Method: You can enable depreciation feature, just selecting either Straight Line or Diminishing Balance method as per your requirement and finally click on Save button.


    7. System Generated Barcode: If this is enabled, barcode generate automatically on adding assets.

      Make sure guests or external users SharePoint site permissions should be provided where the Asset management application is hosted.

    23. Allow User to Book Assets: If this this is enabled, Users can Book Assets.


    24. Select Approvar to Book Assets: If this this is enabled, Users can select approver.


    25. Availability of Assets During Maintenance: If this this is enabled, Users can book asset whcih is in maintenance.


    26. Maintenance Module: If this this is enabled, you can add Maintenance Frequency and Maintenance Due Date while adding assets.

Email Notification


    Here you can setup email notification for assigned or on return assets. You can also setup reminder through email notification for asset expiry and return date of the assets.


    • By default, “To users on assigning assets “is enabled and receives the email notification. if you enter email ID and click on submit both admin and the entered email IDs receives email notification on assigning assets to the users.

    • By default, “To users on returning assets “is enabled and admin receives the email notification. if you enter email ID and click on submit button, both admin and the entered email IDs receives email notification on returning assets from the users.

    • By default, “To groups on assigning assets “is enabled and admin receives the email notification. if you enter email ID and click on submit button, both admin and the entered email IDs receives email notification on returning assets from the group.

    • By default, “To groups on returning assets “is enabled and admin receives the email notification. if you enter email ID and click on submit button, both admin and the entered email IDs receives email notification on returning assets from the group.

    Email Reminder or Enable Automatic Reminders

    By default, “Enable Automatic Reminders” is enabled, admin will receive email notification regarding asset expiry date and returning assets of the users based on days entered in settings.

    Here the person will receive email notification whose email ID(s) entered in “Choose Email Address” box. You can add up to 4 emails (IDs) in choose email ids separated by “;” without space.

    When you enter number as day for “Set Return date Reminder” and click on submit, email notification will receive before the number of days entered to the return date When you enter number as day for “Set Expiry / End date Reminder 1” and click on submit, email notification will receive before the number of days entered to the expiry date of the assets

    In same way “Set Expiry / End date Reminder 2” will works.

    The person who install the application, by default is a admin, such person will receive email notification and email reminder.
