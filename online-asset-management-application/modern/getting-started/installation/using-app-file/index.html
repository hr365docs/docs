  • Installation of Asset Management 365 from App Catalog

  • Prerequisites:

    1. The application can only be installed by a Global Admin or SharePoint Admin by following the steps outlined below.
    2. The application’s features rely on APIs, which can be enabled by a Global Admin with the necessary permissions.
    3. Typically, the link provided will be a OneDrive or similar file-sharing link with a specific expiration date. Please make sure to download the installation package. which is usually a zip or cab file. Once the installation package is downloaded, follow the instructions in the knowledge base to upload the app to the app catalog.

  1. Navigate to the app catalog page using the following URL format:
    https://yourtenantname.sharepoint.com/sites/appcatalog/_layouts/15/tenantAppCatalog.aspx/manageApps

    Replace yourtenantname.sharepoint.com with your actual tenant name, for example: cubiclogics.sharepoint.com.
    Once you are on the page, click on the "Classic Experience" options.



  2. Click on App for SharePoint as shown in below.


  3. This displays the Apps for SharePoint page.

  4. Click on upload.


  5. Choose and upload the file.


  6. Once file got uploaded, select check box and click deploy.

    7. Now you can make this solution or application package available for all sites in the organization, once the application is installed, later you can remove the app from all sites.

    By performing these actions, the selected application package will be downloaded from the Microsoft SharePoint Store into your tenant.


  7. Open your site collection where you need to install your application.

    8. We have to add Asset Management 365 as web part in the page.


  8. Click on New and select Page.




  9. Now click on + icon and then select the Asset Management 365 as shown in below.




  10. When users first access Asset Management 365, a welcome popup is displayed to introduce the platform and guide them through the initial setup. Clicking "Get Started" will initiate the onboarding experience for both administrators and users.


  11. The Quick Setup in Asset Management 365 allows users to configure their asset management system efficiently. The setup includes the following key steps:
    1. Adding Categories
    2. Adding Model Numbers
    3. Adding Vendors
    4. Adding Users
    5. Creating Custom Forms
    6. Configuring General Settings


  12. Add categories to Asset Management 365 to classify assets efficiently and improve asset tracking and organization.



  13. Once a category is added successfully, the status updates to "Completed", indicating the setup step has been finished.


  14. Add model numbers to Asset Management 365 to create a structured and professional tracking system for assets.



  15. After model numbers are added, the setup status changes to "Completed", confirming the task is done.


  16. Add vendors to Asset Management 365 to easily reference suppliers during asset creation and enhance purchasing clarity.



  17. Once vendor details are added, the system marks the status as "Completed", showing successful configuration.


  18. Add users to Asset Management 365 to enable collaboration and improve oversight of asset activities.



  19. When users are added, the status changes to "Completed", confirming the setup is finalized.


  20. Add custom forms in Asset Management 365 to capture department-specific data and support collaborative workflows.


  21. Once custom forms are created, the step is marked "Completed", indicating the customization is in place.

  22. Add general settings to Asset Management 365 to configure global preferences and enable personalized system behavior.




  23. Finally click on Publish button.


  24. Finally, refresh the page.

There are two types of installations:

1. Installation via package upload in the app catalog site.
2. Installation from the SharePoint store (AppSource).

If you delete the package installation and reinstall the app from the SharePoint store, or vice versa, it will not impact or result in data loss. This is because the application's data is stored safely in SharePoint lists, which remain unaffected by the installation method.
