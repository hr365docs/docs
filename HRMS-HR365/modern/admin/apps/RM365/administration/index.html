Administration


In the administration area, users can establish an approval workflow to support process oversight. Only individuals assigned the role of Approver during setup are authorized to complete approval actions within the flow.


By default, the system comes with two pre-created approval workflows: Requisition Approval and Offer Letter Approval. You can set each workflow to Active or Inactive as required.


To begin setting up a new approval process, select the "+New" option from the menu.


Workflow Name: Enter a descriptive name for the workflow.

Job Title: Select the job title applicable to this workflow.

Department: Choose the department responsible for the approval process.

Publish Status: Select the workflow's publication status.

Workflow Description: Provide a short description of the workflow purpose.

In the setup fields, provide the workflow name and description, then select the approval type—either Requisition Approval or Offer Letter Approval.


In each level, you can set whether All or Any approver’s action is required to complete that level.


Then select “Add Approvers” to specify individuals who will be part of the approval process.


Choose the individuals who will serve as approvers, then confirm your selection by clicking the save icon.

  • Level Title: Enter a title for this approval stage.
  • Role: Select the role(s) that should approve at this level.
  • Approver's Name: Select specific users who will approve.
  • User Manager: Include the requester’s direct manager if needed.
  • User Manager's Manager: Include the manager of the requester’s manager if needed.


Auto Approve: Toggle this option on if you want this approval level to be automatically skipped under certain conditions.

  • The level will be skipped if the requester’s name is selected in the conditions.
  • The level will be skipped if the requester’s job title matches the specified criteria.
  • The level will be skipped if the requester’s department matches the specified criteria.



You can assign approval levels, such as L1, L2, and so on, to designate multiple approvers in the process.


If you want the workflow to be active, toggle the Active switch on. Finally, click Save to complete the setup.


To deactivate a workflow, clear the checkbox in the Active section.


The workflow will then move to the Inactive tab.


To reactivate a workflow, select the checkbox in the Active tab.


The workflow will then return to active status.

When creating a requisition, you need to specify the job title, department, publish status .


If the chosen options match an existing approval workflow, that workflow will be triggered automatically.


Based on the selected approver in the workflow, the requisition will be routed to them for approval.
