General Settings


    This section allowing administrators to configure various general preferences such as date formats, themes, languages Currency etc.


    1. Currency Seperator : With this feature, you can change the currency separator by selecting the checkbox for international, India, or no separator. For example, selecting "international" would format the number as 1,000.00, "India" would format it as 1,00,000.00, and "no separator" would format it as 100000.00.


    2. Custom Column : To enable the display of custom columns, turn on the toggle. The custom columns will then be displayed as columns on the homepage and will also be added to the Expense Form.


    3. Expense Future Days :When creating new expenses, the days from today's date up to the selected number of days will be active, while all other days will be disabled. For example, If 5 days is selected and the chosen date is July 17, then the next 5 days, from July 18 to July 22, will be active, while all other days will be disabled.



    4. Display Project Name : To enable the project name, simply turn on the toggle. The project will then be displayed in both reports and the Administration section.



    5. Expense Tracking ID: In the "Expense Tracking ID" field, you can enter the expense ID, such as request or ET, which will be added as a prefix.

Features



    Custom Fields

    The Custom Fields feature allows administrators to create additional fields in the Expense Module. These fields can be used to capture extra information beyond the default system fields.

    Create a Custom Field

    To create a custom field, click on the + Custom Field button, fill in the required details, and then click the Save button. The custom field will then be displayed in the new expense panel.



    Role-based Editability

    Custom columns can also be made editable based on user roles. For example, if the role is Finance Head, that user will be able to add or edit values in the custom column, while other users without this role will only have read-only access.

    1. From the Column Category, select the Expense Module.

    2. Enable the Is Editable option, then fill in the form and select the role (e.g., Finance Head) that should have editing rights.

    3. Click on the Save button to add the custom field.

    4. When creating an expense, the selected field will be editable for users with the assigned role. For all other users, it will remain read-only.

      Integration


        The Integration module allows you to seamlessly connect the Expense Tracker with third-party applications to enhance functionality and automate workflows. One of the key integrations available is with Zoho Books.



        Zoho Books Integration


          Zoho Books is a cloud-based accounting platform that helps businesses manage invoicing, expense tracking, inventory, tax compliance, bank reconciliation, and financial reporting. It reduces manual errors and provides insights for better financial decision-making.

          To enable the integration, go to Integration then click on Account Integration, turn on the Zoho Books toggle this will open the Zoho Books integration panel.


          Step 1: Client ID & Secret
          Log in to https://api-console.zoho.in. Enter the Expense Tracker site URL in both Homepage and Authorized Redirect fields.


          Then navigate to Client secret, Copy the Client ID and Client Secret.


          Step 2: Organization ID
          Log in to https://books.zoho.in, click on the company name, and copy the Organization ID.


          Step 3: Request URL
          Configure Power Automate to obtain the Zoho Books Request URL. In Power Automate, open Solutions, search for HR365 Expense Tracker and open it.


          Then search Zoho and click on edit icon.


          Click on Manual and copy the Http url Paste it into the Request URL field in Expense Tracker.



          Step 4: Redirect URIs
          Enter the Expense Tracker site URL in both Redirect URI fields (Approved and Settings). Click Save to complete setup.


          Mapping Expense Types

          Open the Expense Type panel.


          Select Zoho Books Expense Type.


          Map your master expense type in Expense Tracker to the Zoho Books expense type.


          Expense Synchronization
          After the final approver gives approval, the expense record is marked as Expense Synced in Zoho Books. The details are then visible in Zoho Books.



        AI API Integration


          Azure OpenAI introduces an advanced feature that automatically processes receipts and bills added as attachments. It extracts relevant details, organizes the data, and fills in the required fields, making expense management seamless and efficient. Users can record, categorize, manage, and analyze business expenses with greater accuracy and speed.

          To enable this functionality, navigate to General Settings and open the AI API Integration panel.


          The panel provides configuration options for Azure OpenAI:

          • Enable Azure OpenAI — Toggle to activate or deactivate AI integration.

          • Azure AI Endpoint — REST API or service endpoint hosted on Microsoft Azure.
          • Azure AI API Key — Authentication token to securely connect with Azure AI services.
          • Azure AI Deployment Name — Identifier of the deployed model within your Azure resource.

            • To learn how to generate and retrieve your Endpoint, API Key, and Deployment Name refer this Guide: Configure Azure AI.

          Once configured, click Save to apply the integration settings.


          Expense attachments can then be added directly from the Home Page by selecting Add, or by using the Upload option.


          To enable uploads, toggle on Import Cards in Settings and select Upload from the Home page.


          Select Expense Attachment and upload the required receipts or supporting documents.


          The system will automatically extract the relevant details from the uploaded receipts and populate the necessary fields in the Expense Tracker application.



      • Copilot Integration: Allows enabling or disabling the Copilot assistant feature. To activate, enter the configured Power Automate (PA) URL and click Save. Once activated, Copilot provides intelligent automation and contextual assistance.

        • To integrate Copilot with CLM 365, begin by creating your Copilot agent in Copilot Studio.

        Ensure that you have a Premium Power Automate license to proceed with the configuration.

        In the left-hand sidebar, select Agents.


        If you don’t already have an agent, click on + New Agent to create one, then configure it as required.


        If you already have an existing agent, simply click on that agent to open it.


        In the Knowledge section, add your Knowledge Base.


        Then go to Settings. Under Security, select Web Channel Security and copy the Secret 1 key.




        Next, in Power Automate, go to My Flows and import the Copilot server solution.


        Open the flow and click Edit.


        Disable the New Designer toggle and paste the previously copied Secret 1 value into the Headers field.


        Save the flow and copy the HTTP GET URL generated under the When an HTTP request is received trigger.


        Return to the application, enable the Copilot integration toggle, and paste this URL into the configuration field.
