Workflow


    Note: Unlicensed user creation in O365 works on Power Automate setup

    1. HR Manager: The one who can approve pre joining request.
    2. Admin: They just notified about to assign resources to new joinee.
    3. IT Teams: They just notified about to assign resources to new joinee.

      4. We can setup these Teams in advance settings

    Roles in the application:
    1. Admin:

      2. Admin have full control; they can add and manage users for Onboarding application.

      1. Can add roles, cost center, departments, and grades.
      2. Can be able to change settings of the application.
      3. Can be able to do pre joining activities.
    2. HR:
      1. The person with role HR can add and manage users for application.
      2. Can do Onboarding process activities.
    3. User:
      1. The person with role user can use only employee self-service portal.
    4. No Access:
      1. The person with no access role cannot access onboarding application.
