    Features

    The Features section provides advanced tools such as custom fields, cost centers, departments, employee grades, mileage tracking, and approver budget limits to enhance expense management efficiency and flexibility.


    Custom Fields

    The Custom Fields feature allows administrators to create additional fields in the Expense Module. These fields can be used to capture extra information beyond the default system fields.


    Create a Custom Field

    To create a custom field, click on the + Custom Field button, fill in the required details, and then click Save. The custom field will then be displayed in the new expense panel.



    Role-based Editability

    Custom columns can also be made editable based on user roles...

  • From the Column Category, select the Expense Module.

  • Enable the Is Editable option, then fill in the form and select the role (e.g., Finance Head) that should have editing rights.

  • Click on the Save button to add the custom field.

  • When creating an expense, the selected field will be editable for users with the assigned role. For all other users, it will remain read-only.

    Approver Budget Limit

    Approver Budget Limit is as per Approver approved expense amount is validated or exceed in expense tracking applications.

    • Approver Budget limit add for particular approver they can approve this much amount only .

    Navigate to 'General Settings' and click on 'Budget Limit'."


    Approver Budget limits: A toggle to activate or deactivate the Approver Budget limits integration in the application.
    Save Button: A button to save changes or configurations made in the Approver Budget limit settings


    Navigate to Administration, then select Approver Budget.


    Click on Approver, and a panel will open displaying Frequency, Approver Budget, Start Date, and End Date. You can define the budget limit and specify its applicability. Additionally, you can add multiple approvers as needed.



    Click on Approver, and a panel will open displaying Frequency, Approver Budget, Start Date, and End Date. You can define the budget limit and specify its applicability. Additionally, you can add multiple approvers as needed.


    After saving, the Approver Limit is added to the Detail List. Whenever an approver approves an expense, the amount will be automatically updated in the list.


    1. When an expense is approved, the Available Amount and Consumed Amount are updated accordingly.

    2. there are multiple approvers at the same level, only the approving approver’s amount is updated, while the other approver’s amount remains unchanged until they approve.


    When an expense is created, it is sent for approval based on the defined approval workflow.


    After creating an expense, the amount is deducted from the approver's limit. For example, if the approver's initial limit is 10,000 and an expense of 1,000 is approved, the Consumed Amount becomes 1,000, and the approver can approve up to the remaining 9,000.


    After the 1st level approval, the 2nd level approvers' amounts will also be updated based on the total approved amount, following the same process as the 1st level approver.

      Delete Expense

      Deleting an Expense makes it inactive, and it can be restored based on the expiration date.
      As per settings, the Recycle Bin and Bulk Inactive options will be available. These features are only applicable to Finance, Finance Head, and HR Head.

      Navigate to General Settings and click on Delete Expense.


      A dialog box will open.


      Delete Expense: A toggle to activate or deactivate the Delete Expense feature, which will reflect across the entire application.
      Delete Expense After 30 Days: Any inactive expense will be automatically deleted after 30 days through Power Automate.
      Save Button: A button to save any changes or configurations made in the Delete Expense settings.


      After that, you will be able to see the Recycle Bin and Bulk Delete options.


      After clicking the Delete icon.


      Click on Confirm, and the selected expenses will be marked as inactive and moved to the Recycle Bin.


      A panel will open for the Recycle Bin, where all the deleted expenses will be displayed.


      When deleting any expense, if the expense has an associated budget limit, the corresponding amount should be reverted (refunded) to the respective Department, Location, Cost Center, and Project.


      After reviewing the expense, click on the Restore icon to bring it back.

      Expense Future Days

      When creating new expenses, the days from today's date up to the selected number of days will be active, while all other days will be disabled. For example, If 5 days is selected and the chosen date is July 17, then the next 5 days, from July 18 to July 22, will be active, while all other days will be disabled.




        Logs Types

        After enabling log collection, the user will see an option for 'Log Types' below."


        • Debug Logs: Captures detailed system activity for in-depth troubleshooting. It generates extensive logs and requires significant storage space. Ideal for resolving specific issues.

        • Information Logs:Records all application actions, providing a complete activity trail. Useful for analyzing suspected issues but consumes moderate storage.

        • Warning Logs:Logs errors and potential risks that might not immediately disrupt functionality. Focused on early issue detection with minimal storage impact.



        If the user selects 'Debug' from the log types and saves it.


        Then, the user will see an option to 'Download Logs' after going to the help page.


        In the field, the current user's name must be entered, and then by clicking on 'View,' the user can access and view the debug logs.



        The user can click on the 'Share' icon to share the logs. Encrypted logs will be shared with Cubic for further troubleshooting and to provide a more efficient resolution.


        For downloading, the user can click on the 'Download' option to retrieve the logs.
