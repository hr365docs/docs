General Settings


    This section allowing administrators to configure various general preferences such as date formats, themes, languages Currency etc.


    1. Theme: In this section, you can switch the theme to dark, light, or a Site theme color.


    2. Date Format : In the Date Format section, you can customize the date format to options such as MM-DD-YY, YY-MM-DD, or any other format you prefer.


    3. Currency : With this feature, you can change your currency to options such as USD, EUR, INR, and more.


    4. Currency Seperator : With this feature, you can change the currency separator by selecting the checkbox for international, India, or no separator. For example, selecting "international" would format the number as 1,000.00, "India" would format it as 1,00,000.00, and "no separator" would format it as 100000.00.


    5. Language : With this feature, you can change the language selection if the browser's default language is not detected. You can select the desired language from a dropdown menu, and that language will be applied to your application.For example, if you select French,the application language will change to French.


    6. Custom Column : To enable the display of custom columns, turn on the toggle. The custom columns will then be displayed as columns on the homepage and will also be added to the Expense Form.



    7. Expense Future Days :When creating new expenses, the days from today's date up to the selected number of days will be active, while all other days will be disabled. For example, If 5 days is selected and the chosen date is July 17, then the next 5 days, from July 18 to July 22, will be active, while all other days will be disabled.



    8. Display Project Name : To enable the project name, simply turn on the toggle. The project will then be displayed in both reports and the Administration section.



    9. Expense Tracking ID: In the "Expense Tracking ID" field, you can enter the expense ID, such as request or ET, which will be added as a prefix.



    10. Sharepoint Page Configuration:

      1. Remove Custom Webpart CSS :
        2. Activating this toggle option within the settings will result in the automatic removal of empty space surrounding the designated webpart.

      2. Hide Webpart Title on SharePoint Page :
        3. This functionality enables users to conceal the titles of webparts on SharePoint pages where applications are active.

      3. Hide Side Navigation Panel on SharePoint Page :
        4. Hide side navigation panel in Sharepoint page where App is running.

      4. Hide Top Command Bar on SharePoint Page :
        5. Toggling this off will hide the command bar at the top of the SharePoint page.

      5. Hide Top Bar on SharePoint :
        6. Turning it off will hide the top bar in SharePoint.

      6. Hide Comment Container:
        7. Toggling it off will hide the comment container in SharePoint.

      7. Hide Top Site Header :
        8. Turning it off will hide the top site header.

User Roles and Permissions


    The admin can create new users and assign roles and permissions using this feature.


    1. Admin :
      - Full Settings Access
      -Access to all settings and features.
    2. Finance :
      - Limited Access
      - Allow access to the panel to user management only.
    3. HR :
      - Limited Access
      - Access to user management only.
    4. Finance Head :
      - Customizable Settings
      - Date Format : Ability to set and modify the date format.
      - Currency : Control over currency settings.
      - Currency Separator : Modify currency separator settings.
      - Display Custom Column : Customize columns in various displays.
      - Expense Future Days : Set and manage future expense days.
      - Display Project Name : Control the visibility of project names.
      - Email Templates : Manage and edit email templates.
      - Users : Manage user accounts and roles.
      - Custom Fields : Create and manage custom fields.
      - Cost Center : Manage cost centers.
      - Departments : Manage department settings.
    5. HR Head :
      - Customizable Settings
      - Date Format : Ability to set and modify the date format.
      - Display Custom Column : Customize columns in various displays.
      - Display Project Name : Control the visibility of project names.
      - Email Templates : Manage and edit email templates.
      - Users : Manage user accounts and roles.
      - Custom Fields : Create and manage custom fields.
      - Cost Center : Manage cost centers.
    6. User :
      - No Access : No access to user to any settings .
      - No Permissions : No permissions to user any settings .

    Expense Type Limits Based on Grade
    Users can set a limit for expense types based on employee grades. For example, if a user's grade is E1, and a limit of 2000 is defined for the "Per Diem" expense type, the user can create expenses under this type only within the specified limit of 2000.


    While creating a Per Diem expense, if the amount exceeds the defined limit, an error validation message will be displayed.


Approver Budget Limit is as per Approver approved expense amount is validated or exceed in expense tracking applications.

  • • Approver Budget limit add for particular approver they can approve this much amount only .

Navigate to 'General Settings' and click on 'Budget Limit'."


Approver Budget limits: A toggle to activate or deactivate the Approver Budget limits integration in the application.
Save Button: A button to save changes or configurations made in the Approver Budget limit settings


Navigate to Administration, then select Approver Budget.


Click on Approver, and a panel will open displaying Frequency, Approver Budget, Start Date, and End Date. You can define the budget limit and specify its applicability. Additionally, you can add multiple approvers as needed.



Click on Approver, and a panel will open displaying Frequency, Approver Budget, Start Date, and End Date. You can define the budget limit and specify its applicability. Additionally, you can add multiple approvers as needed.


After saving, the Approver Limit is added to the Detail List. Whenever an approver approves an expense, the amount will be automatically updated in the list.


1. When an expense is approved, the Available Amount and Consumed Amount are updated accordingly.

2. there are multiple approvers at the same level, only the approving approver’s amount is updated, while the other approver’s amount remains unchanged until they approve.


When an expense is created, it is sent for approval based on the defined approval workflow.


After creating an expense, the amount is deducted from the approver's limit. For example, if the approver's initial limit is 10,000 and an expense of 1,000 is approved, the Consumed Amount becomes 1,000, and the approver can approve up to the remaining 9,000.


After the 1st level approval, the 2nd level approvers' amounts will also be updated based on the total approved amount, following the same process as the 1st level approver.

    Delete Expense

    Deleting an Expense makes it inactive, and it can be restored based on the expiration date.
    As per settings, the Recycle Bin and Bulk Inactive options will be available. These features are only applicable to Finance, Finance Head, and HR Head.

    Navigate to General Settings and click on Delete Expense.


    A dialog box will open.


    Delete Expense: A toggle to activate or deactivate the Delete Expense feature, which will reflect across the entire application.
    Delete Expense After 30 Days: Any inactive expense will be automatically deleted after 30 days through Power Automate.
    Save Button: A button to save any changes or configurations made in the Delete Expense settings.


    After that, you will be able to see the Recycle Bin and Bulk Delete options.


    After clicking the Delete icon.


    Click on Confirm, and the selected expenses will be marked as inactive and moved to the Recycle Bin.


    A panel will open for the Recycle Bin, where all the deleted expenses will be displayed.


    When deleting any expense, if the expense has an associated budget limit, the corresponding amount should be reverted (refunded) to the respective Department, Location, Cost Center, and Project.


    After reviewing the expense, click on the Restore icon to bring it back.


    User Role scope


    User Role Scope is used to filter expenses based on Location, Department, and Cost Center in the expense tracking application.

    1. As per the role scope, the expenses will be displayed in the Expense Home Module.
    2. This feature is applicable only to Finance, Finance Head, and HR Head.

    Navigate to User Roles and Permissions and click on Role Scope.


    Navigate to User Roles and Permissions and click on Role Scope.


    Enable Role Scope: A toggle to activate or deactivate the Role Scope feature, which will reflect across the entire application.
    Save Button: A button to save any changes or configurations made in the Enable Role Scope settings.


    Then, click on the User Module, add or edit a user, and select the role as Finance, Finance Head, or HR Head. The Role Scope field will only be displayed for these roles.


    The Role Scope has three options: ALL, Cost Center, Department, and Location.

    1. If you select Location, the Location Scope field will appear below.
    2. If you select Department, the Department Scope field will appear.
    3. If you select ALL, all fields will be displayed.


    After saving, check the Expense Home Page. For example, if Joni's Role Scope is set to Location and the Location Scope is set to Canada, then, when Joni is the current user and their role is Finance Head, only expenses related to users who have Canada as their location will be displayed in the Expense Home and Approved file.


    Following the set flow, expenses will be displayed based on the defined Role Scope and Location Scope settings.


    When an expense is created, it will be checked against the current user’s Role Scope (Location, Department, or Cost Center) to determine which expenses should be displayed based on the user’s assigned scope.



    When an expense is created, it will be displayed for the current user only if the Reported By field matches their Role Scope settings (Location, Department, or Cost Center).
