Settings

In settings page there are four tabs namely general, notification, features and advanced settings.

  1. General

    1. Date Format: If you select date format such format would be reflect in whole application.

    2. Default currency: If you change default current will be reflect in currency field of in add project page.

    3. Rename Display Title: If you want to rename instead of client title you can do it. You can see renamed title in place of client in whole application.

    4. Follow ISO week number format: If this is enabled ISO week number format will enable

    5. Show pending weekly timesheet from: if this feature is enabled, you can see pending weekly timesheet option under timesheet from side panel.

    6. Brand logo: if you setup brand logo, you can be able to see it at the top of the side panel

    7. Predefined Themes: You can change theme color of the application from predefined themes.

    8. Display: Display of fields like activities, project ID, task ID, department ID and task can be controlled.

      • Activities: If it is enabled, you can see in navigation panel under administration from where you can add various activities. It will also display in weekly timesheet, track time (Please enable track time from advance setting under feature than track time tab is visible in-home page, by default it is disabled), daily report, weekly report table under reports.

      • Project ID: If it is enabled, it will display daily report and weekly report and monthly report's table under reports.

      • Task ID: If it is enabled, it will display daily report and weekly report's table under reports.

      • Department ID: if it is enabled, it will display in daily report, weekly report and monthly report's table under reports.

      • Display Task: if it is disabled task status tab of home page and task column of all tables in application will be disappear or hide.

  2. Notification

    1. Reminder on last working day of the week: If it is enabled, user get notified about submission of timesheet on last working day of the week. When you enable it, “Select last working day of the week” dropdown will appear it select day and finally click on save settings.

    2. Task assign notification: By default, it is enabled, you can disable it, users will not be notified about tasks when you assign to them.

    3. Auto approve notifications: By default, this is disabled, if you enable it, for auto approve projects, user get notified through mail about approval of his timesheet.

    4. Timesheet submission and approval notifications: By default, it is enabled, if you disable it, users will not be notified about submission of their timesheets and also approval or rejection of timesheets.

    5. Notification to Project manager in case overdue: If this feature is enabled project manager will notify or remind about their user's submission of timesheets in case of overdue.

    6. Following day as another notification: If this feature is enabled, users will get notify or reminds about submission of timesheets on next day of last working day of week.

    7. Weekly report overdue submission day: When you add days and enabled it, users will get notify or reminds about the submission of their timesheets in case of overdue.

  3. Features

    1. Work Place: If it is enabled users can choose their work place while submitting daily or weekly timesheet.

    2. Mandatory task selection while tracking time: If it disabled, users could submit timesheet without selecting tasks.
    3. Self-Assign Task: If it is enabled, Task tab in will display form where user or employee can create their task itself.

    4. Allow self-assign task billable: If it is enabled, users can create billable task.

    5. Allow task time tracking after deadline: If it is enabled users can track the time or submit timesheets after task deadline date.

    6. Allow all user to track time for all projects: If this is enabled, users those have been added up in timesheets application are allowed to submit timesheets for all projects thou they are not a team members added in projects.

    7. Allow timesheet submission for upcoming days in current week: If this is enabled, users are able to save or submit timesheets for upcoming days of current week in advance.

    8. Allow user to submit timesheet for weeks: Here you can add number, so that user can submit timesheets for past weeks. Example If you add 2 or 3 number it means that users are allowed to submit timesheets of past 2nd or 3rd week.

    9. Timesheet Approved By: If you select as project manager, project manager will receive timesheet about approval from user's submission. If you select people manager, people manager will receive timesheet about approval from user submission (admin can change people manager of user from add user page).

    10. Mandatory tasks start date and end date: we make project managers or self-assign tasks by users to create tasks with task start date and end date.

    11. Allow Admin to edit approved timesheet: If this is enabled, edit icon will display in home page for weekly timesheet tab for admin, from where admin can edit submitted or approved timesheets of users

    12. Enable program manager in project creation: If it is enabled, program manager field will display in add project page from where can add program manager of the project.

    13. Enable onsite billing: If it is enabled, “Individual with Onsite Rate” field will display in add project page under billable.

    14. Restrict daily and weekly hours: If we setup limits hours for daily and weekly, users are not allowed to put time behind limits for daily and weekly timesheets.

    15. Daily time tracking: If it is enabled, Track Time tab will display in home page from where we can submit daily timesheets.

    16. Enable daily timesheet report notification: If it is enabled, users get notified about their submitted daily timesheets or track time
  4. Advanced Settings
    1. a. Time tracking delegation by Admin / PM: If it is enabled log weekly as dropdown will display when you click on weekly timesheet or track time tab in home page, from where admin and project manager can submit timesheet behalf of users.
    2. b. External users: If it enabled and provide site permission for externals, admin can add users to this application.
    3. c. Integrate Time Off Manger Plus with Timesheet Plus: If it is enabled, Time off option will display in side panel from where you can submit leaves

      Notes

      • • Users not able to put time for leave days.
      • • It will work only when you install HR365 Time off Manager Plus and Timesheets plus application in same SharePoint site and should be external date list (By default setup is external database, no need to change)

    4. d. Database path: You can setup database outside the hosted add-in. Please enter the SharePoint site/subsite location where SharePoint list is used to store the data.
    5. e. Sync Exchange / AD Properties to Timesheet Plus: If it enabled, graph API credential fields appears and if you put those and submit, exchange and AD properties will appear in SPO/EXO User Profile Properties column of add custom fields page
    6. f. Time Tracking Days: Users are cannot put time for weekly timesheet or track time for day which are disabled. You can disable weekend days from this feature.
    7. g. Auto approve time sheet: The selected users from this option when they submit the timesheet, their timesheet will approve automatically. This is user specific auto approve timesheets.
