General

    General Settings

    General settings can be done by only admin role person. By default, the one installed application is an admin, you can assign admin tole to person from the user’s option in general settings page.

    Custom Settings: From here you can theme color, date format, rename display title for teams and company name, save link validation, Integrate Asset Management Plus with Helpdesk Plus, Graph API Integration and company or brand logo.

    • Date format: If you change the date format, it will update in entire application. • Display title for teams: If you change the title, it will update in entire application. You can find update display title just below the user’s option from where you can create departments or groups. The created one will display in new ticket panel which can be selected during ticket submission. • Company name: If you update company name, it will update in closed and resolved emails notification. • Save link validation: Here you can add the number of the days to the survey link expire. • Tickets can be deleted by: You can allow to delete the ticket other roles person from here. • Integrate Asset Management Plus with Helpdesk Plus: You can integrate HR365 Asset Management application with helpdesk plus application, make sure both applications should be installed in same site location. Once you integrate you will find assigned asset to user in comment page as shown in below. you can click on ticket id in home page to view the comment page. • Graph API integration: If you configure graph API, Ticket mailbox works with help desk email ID. • Use teams code as ticket suffix: Just enable the toggle switch for Use Teams code as ticket suffix. • Display resolved tabs in home page: Here you can enable or disable all resolved related tabs in home page. • Brand logo: If you update your company logo, it will update at top of side panel as show in below image.

    Users: Admin can assign roles to O365 users with ticket scope as per requirement in Help Desk application. You can select both supervisor and agents for multiple teams but ticket scope will remain same and roles one. What would happen if user role is agent and he/she is supervisor of team? They will receive notification as a part of supervisor of teams according to Email notification settings and perform activities as per agent roles.
