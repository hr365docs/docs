Webhook


  • A webhook is a powerful tool used for real-time communication between different applications or systems.The data posted by the user is initially collected and held in the system, but it is only stored at the specified webhook URL once the associated task is completed. This means that the webhook acts as a storage point, capturing the data only after all necessary processes or actions are finished. Once the task is completed, the data is then securely stored in the webhook URL, ensuring that it is properly organized and ready for further processing, integration with other systems or applications as required. This ensures that the data is accurately handled and retained only after all necessary conditions or actions have been met.




  • To set up a webhook, the first step is to select a webhook name. Once the name is chosen, you need to choose a specific task process, which includes options such as Onboarding, Offboarding, Offboarded, and Pre-Onboarding. The webhook will then fetch all the tasks related to the selected task process, allowing for seamless integration and data retrieval. Additionally, you can select up to five webhook URLs to receive the data, with the flexibility to edit or delete these URLs as needed. This setup ensures that the appropriate systems are notified and updated based on the selected task process.



    • The data posted in the webhook url will be in the below format
    {TaskDetials: {}, UserDetails: {}}

Sharepointhook


  • When data is submitted through a form, the system ensures that each piece of data is mapped to the correct field or column in the target SharePoint list or database. This ensures consistency and proper organization of the information across different systems or forms. Additionally, once data is posted to the backend, a new column may be created automatically. After the task is completed, this data is then mapped to the appropriate column in the SharePoint list, ensuring that the information is correctly aligned and accessible for further use or analysis.




  • To set up a sharepointhook, the first step is to select a Title. Once the Title is chosen, you need to choose a specific Task process, which includes options such as Onboarding, Offboarding, Offboarded, and Pre-Onboarding. The sharepointhook will then fetch all the tasks related to the selected task process, allowing for seamless integration and data retrieval. Additionally, you can select Target List ,Once the task is completed the data will get mapped to the particular column created in the List.

AI Integration


Azure OpenAI within EO365 enhances course creation and employee training by automatically generating introductions, questions, and chapters. This eliminates manual effort, ensures grammatical accuracy, and accelerates the onboarding process with smart AI assistance.


To enable this functionality, navigate to General Settings and open the AI API Integration panel.


Inside the panel, configure the following options for Azure OpenAI:

  • Enable Azure OpenAI — Use the toggle to turn AI integration On or Off.

  • Azure AI Endpoint — The REST API or service endpoint hosted on Microsoft Azure.
  • Azure AI API Key — Authentication token required for secure connection with Azure AI services.
  • Azure AI Deployment Name — Identifier of the deployed AI model in your Azure resource.

    • To learn how to generate and retrieve your Endpoint, API Key, and Deployment Name refer this Guide: Configure Azure AI Documentation.

After entering the required details, click Save to activate the integration.


Courses

  • Add Course Details — Enter the course name and description, then click Next to continue.


  • Generate Course Introduction — Click AI Generate to automatically create a professional introduction for the course.


  • Refine & Correct — Add your own edits, or click AI Generate again to polish and fix grammatical errors.




Questions

  • Generate with AI — Instead of manual entry, click Generate Questions using AI.


  • Set Parameters — Define type, section, answer format, difficulty, number of questions, and time limit, then click Generate.


  • Review Questions — AI generates questions with options and correct answers based on your inputs.


  • Select & Customize — Choose required questions, edit as needed, or select all for quick setup.


  • Save — Finalize by clicking Save or Save as Draft.



Chapters

  • Generate with AI — Click Generate Chapters using AI instead of creating them manually.


  • Set Parameters — Define type, section, difficulty level, number of chapters, and time limit, then click Generate.


  • Review Chapters — AI generates structured chapters based on your prompt, which can be refined or customized.


  • Save — Store your work by clicking Save or Save as Draft.
