General Settings

The General Settings section allows administrators to configure key preferences such as date format, navigation theme, default language, branding elements, and system-level integrations for SharePoint and Microsoft Teams.

Date Format

  • Adjust the Date Format according to your organizational preference. The selected format will be applied across all modules and interfaces within the application.

Side Navigation Theme

  • Choose from available theme options to personalize the application interface. Supported modes include Light, Dark, and Site Theme Color.

GCC Tenant

  • If your Microsoft 365 tenant is a GCC (Government Community Cloud) tenant, enable this setting to ensure compatibility and secure integration with Microsoft cloud services.

Show Fav Icon

  • Toggle this option to display the Favicon on the browser tab, allowing users to easily identify the application when multiple tabs are open.

Default Language

  • The Default Language setting allows administrators to define the primary language for the application interface. Users can either choose to automatically apply their browser’s default language or manually select a preferred language from the dropdown list.

  • Language selection as browser default language: When this toggle is enabled, the application adopts the language defined in the user’s browser settings.

  • Language Selection: Users can manually select a specific language (e.g., English) from the dropdown if they prefer not to use the browser default option.

Use SharePoint URL for MS Teams Installed Application

Enable this setting to integrate SharePoint pages directly into the Microsoft Teams application. This allows users to access SharePoint resources seamlessly from within Teams.


To configure, toggle on the “Use SharePoint URL for MS Teams installed application” setting and provide your SharePoint site URL in the following format:
https://yourtenantname.sharepoint.com/sites/SiteName/SitePages/PageName.aspx
After entering the URL, click Save to apply changes.

Hide SharePoint Page's Default Components

  • Activate this option to remove unnecessary empty spaces and hide default SharePoint webpart titles for a cleaner interface.


  • You can also hide the side navigation panel on SharePoint pages where the app is embedded.

    logs collection

    The user has the option to enable log collection. If you want to collect logs, please turn on the toggle. These logs are used for troubleshooting any issues occurring in your application.

    Logs Types

    After enabling log collection, the user will see an option for 'Log Types' below."


    • Debug Logs: Captures detailed system activity for in-depth troubleshooting. It generates extensive logs and requires significant storage space. Ideal for resolving specific issues.

    • Information Logs:Records all application actions, providing a complete activity trail. Useful for analyzing suspected issues but consumes moderate storage.

    • Warning Logs:Logs errors and potential risks that might not immediately disrupt functionality. Focused on early issue detection with minimal storage impact.

    • Error Logs:Logs critical errors affecting application functionality. Provides essential details for immediate troubleshooting with minimal storage usage.


    If the user selects 'Debug' from the log types and saves it.


    Then, the user will see an option to 'Download Logs' after going to the help page.


    In the field, the current user's name must be entered, and then by clicking on 'View,' the user can access and view the debug logs.


    The user can click on the 'Share' icon to share the logs. Encrypted logs will be shared with Cubic for further troubleshooting and to provide a more efficient resolution.


    For downloading, the user can click on the 'Download' option to retrieve the logs.

Naming Convention

  • Configure Naming Conventions to standardize labels and headings across the Employee Self-Service (ESS) portal for consistent document categorization.


  • Defined naming sections appear as document categories under which users can upload or manage related files.

Company Details

  • Manage and update organization-specific details including company name, tagline and company info. This information will appear in various system-generated reports and templates.

Brand Logo

  • Upload and display the organization’s Brand Logo on SharePoint and ESS pages. You can also hide or adjust placement to align with your branding preferences.
