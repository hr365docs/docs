    • Custom Columns

      The Custom Columns feature allows administrators to create additional data fields that extend the default onboarding forms and records. These fields can be added to both employee and project forms to capture organization-specific information.


    • Click on the + Column button to create a new custom field.

    • Provide the Field Name, choose the Column Data Type based on the data you wish to store (for example: text, number, date, etc.).


    • Click Submit to save the column. The newly added field will appear in the respective form section, enhancing the flexibility of data entry and management.



      • This functionality helps organizations customize their onboarding data capture process, ensuring that all relevant details specific to their workflows are maintained.

    Condition Based Column

      The Condition Based Column feature allows administrators to dynamically control the visibility of form fields based on specific conditions. This ensures that employees only see fields relevant to their role or employment type during onboarding.

    • Click on the + Condition Based Column button to configure visibility rules for existing fields.


    • Then click + Conditions to define new conditional logic.


    • Select the relevant Form (e.g., Preonboarding) and choose a Condition Field such as Employee Type.


    • For each available option (for example, Permanent, Contractual, Consultant), specify the fields that should appear when that condition is met using the Select Fields to Show Condition dropdown.


    • Once all conditions are defined, click Save to apply the settings.


      • This capability ensures a dynamic and user-friendly onboarding experience — only relevant form sections are displayed to users, improving accuracy and simplifying data entry.

    Quick Share Links

    • Click +Add to create new Quick Links.



    • Fill in the required fields and click Save to apply the changes.


    Approval Workflow

      Configure approval workflows to streamline your onboarding or pre-onboarding processes.

    • Click on Approval Workflow to proceed.



    • Click +Add to add a new approval workflow.



    • Fill in the fields as required and click the Save icon to complete setup.


    • A dedicated Pre-Onboarding Approval Workflow has been added, allowing approvers to review candidate details before transitioning to the onboarding stage.



    Delegation

      The Delegation feature allows administrators or users to assign their onboarding tasks temporarily to another user, ensuring process continuity.

    • Open the Delegation panel to view or create delegation entries.



    • Click +Add to create a new delegation entry.



    • Admins can create delegations for all users; standard users can only delegate their own responsibilities.



    • Select the delegatee and define start and end dates for the delegation period.





    • Use the Active toggle to enable or disable delegation, then click Submit to save changes.

    Guest Users

    • The Display Guest User toggle controls the visibility of guest user information within your system.


    Assessments

      Organizational assessments help identify employee strengths and development areas, supporting continuous improvement and strategic growth.


    Course

      Enable the Course toggle in the settings to activate the course management feature. This functionality allows administrators to create, manage, and assign learning courses to employees, supporting continuous development and skill enhancement. Once enabled, the created courses will be visible in the Employee Self-Service Portal, where employees can access course materials, monitor progress, and complete assigned training.


    Training

      Enable the Training toggle in the settings to make the training section visible within the application. Once enabled, administrators can create and manage training programs for employees, helping them enhance their knowledge and professional development. The assigned trainings will be displayed in the Employee Self-Service Portal, allowing employees to access and complete their learning activities efficiently.


    Show Home Sample Data

    • Enabling the Show Home Sample Data toggle displays sample widgets and analytics on the home page, helping users visualize layouts before real data is loaded.




    Preonboarding

    • The Make Pre-Onboarding Visible toggle controls visibility of the pre-onboarding process in your system.

    • The Approving Pre-Onboarding Workflow feature enables designated users to review and approve new hire details before formal onboarding.



    • The Enable Position in Home Page option allows you to display onboarding positions directly on the home dashboard for easier tracking and visibility.


    • The Pre-onboarding Position Approval setting allows teams to approve newly created positions. Simply choose the team responsible for the approval process in the Positions module.
