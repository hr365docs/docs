    General Settings

    The General Settings section in Revenue365 provides fundamental configuration options for system customization. It includes settings related to appearance, localization, and branding. Users can adjust preferences such as themes, date formats, and currency. Additionally, it allows customization of SharePoint components. These settings help personalize the user experience and maintain consistency across the platform.


    Theme

    Revenue 365 Application has the theme option where the user can select Dark Mode, Light Mode, or Site Mode. The theme will then change accordingly. Users can also switch between modes based on their preference for better visibility and comfort.



    Date Format

    Users have the ability to configure the date format globally through this option. Should the user choose the MM-DD-YYYY format, it will be consistently applied throughout the Revenue 365 application.


    Default Currency

    Similar to the date format, the user can select a default currency. For example, if the user selects USD as the default currency, it will be applied consistently across the Revenue 365 application for transactions, reports, and all financial data.




    When changing the default currency, a dialog box will appears: Selecting Confirm updates the currency and deletes all report data; Cancel aborts the action.


    Langauge

    The default language is English, but you can change it to your preferred language.


    Sharepoint page components



    Remove Custom Webpart CSS: Enable this toggle will remove space outside of Webpart.


    Hide Webpart Title on SharePoint Page: Enable this toggle will remove the Webpart title.

    Hide Top Command Bar on SharePoint Page: To Enable this toggle you can hide Command bar from sharepoint page.


    Hide Top Bar on Sharepoint:to Enable this toggle you can hide top sharePoint nav header.


    Hide Comments Wrapper: Enable this toggle will hide comments wrapper in sharepoint page where app is running.

    Sample Dashboard Data

    When the Sample Dashboard Data toggle is enabled, sample data is displayed on the dashboard.



    Clicking on "Click here" will replace the sample data with the original dashboard data.


    Revenue 365 Sample Data

    When the Revenue 365 Sample Data toggle is enabled, sample data will appear on the homepage view.



    Language Selection as a Browser Default Language

    When the user enables the "Browser Default Language" option, the application will automatically use the browser's default language.


    Enable logs collection

    The user has the option to enable log collection. If you want to collect logs, please turn on the toggle. These logs are used for troubleshooting any issues occurring in your application.

    Logs Types

    After enabling log collection, the user will see an option for 'Log Types' below."


    • Debug Logs: Captures detailed system activity for in-depth troubleshooting. It generates extensive logs and requires significant storage space. Ideal for resolving specific issues.

    • Information Logs:Records all application actions, providing a complete activity trail. Useful for analyzing suspected issues but consumes moderate storage.

    • Warning Logs:Logs errors and potential risks that might not immediately disrupt functionality. Focused on early issue detection with minimal storage impact.

    • Error Logs:Logs critical errors affecting application functionality. Provides essential details for immediate troubleshooting with minimal storage usage.


    If the user selects 'Debug' from the log types and saves it.


    Then, the user will see an option to 'Download Logs' after going to the help page.


    In the field, the current user's name must be entered, and then by clicking on 'View,' the user can access and view the debug logs.



    The user can click on the 'Share' icon to share the logs. Encrypted logs will be shared with Cubic for further troubleshooting and to provide a more efficient resolution.


    For downloading, the user can click on the 'Download' option to retrieve the logs.

Use Sharepoint URL for MS Teams installed application

The option Use SharePoint URL for MS Teams installed application enables the integration of SharePoint links within the MS Teams application, allowing users to access SharePoint resources directly from Teams.


Enable the “Use SharePoint URL for MS Teams installed application” toggle, then enter your SharePoint site URL in the following format:
https://yourtenantname.sharepoint.com/sites/SiteName/SitePages/PageName.aspx
After entering the URL, click the Save button to apply your changes.


    Brand logo

    Here you can add your company logo, which will be displayed at the top of the navigation bar. From here, the admin can add or update the company logo. The updated logo will appear at the top of the navigation bar. The recommended size for the logo is 150 X 60, with the same aspect ratio, and a transparent or white background is recommended for better visibility and user experience.


    Click on the 'Upload Logo' button, then select the logo file. After selecting, the user will have the option to crop the logo, allowing them to adjust the image as needed.


    After cropping, the user can click on the 'Save Logo' button. Once saved, the logo will be visible on the top bar.



    If the user wants to remove the logo they added and upload a new one, they can click the 'Reset Logo' button.
