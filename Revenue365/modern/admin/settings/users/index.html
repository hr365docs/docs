    User, Roles And Permissions


    Add/Edit Users: Allows administrators to add or modify user accounts within the system. Administrators can manage access levels, assign roles, and define permissions for each user.


    A user can be assigned different roles for different applications.
    These roles define the level of access and permissions granted within each application.

    • Admin – Manages system configurations, user roles, and overall platform settings.
    • Invoice Manager – Manages invoice creation, submission, and financial reconciliation.
    • Invoice Approver – Reviews and approves submitted invoices for further processing.
    • Coordinator – Handles the coordination of team activities, schedules, and task progress.
    • User – Has limited access for task execution, assigned workflows, or viewing data.


    Full Sync for Users: This feature allows administrators to synchronize all user data from an external directory or source (such as Microsoft 365) into the system. It ensures that user information like names, emails, and roles are up to date and accurately reflected in the application.


    Update from M365: This option allows administrators to refresh or update existing user details from Microsoft 365. It syncs only the latest changes (such as name, email, or role updates) for already added users without performing a full sync.



    Clicking on a username allows you to edit user details, including changing the status from Active to Inactive.


    After making the change, click on Update. The user will then appear under the Inactive section.
