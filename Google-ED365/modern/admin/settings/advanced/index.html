  2. Analytic report

    3. The Analytics Report is available under the Advanced section. It offers two distinct views to analyze user activity:


    1. User-Based View

    This view displays user-specific data by default. It highlights the components each user has accessed, along with the total time spent on each.

    • • Clicking on a user (e.g., Isaiah Langer) displays detailed activity logs for that specific user, showing all components they have interacted with.



    • Clicking on a component within the user's record provides individual usage details for that component.


    2. Component-Based View

    This view displays a list of all components accessed by users.

    Clicking on a component reveals detailed analytics, including:

    o Users who accessed the component
    o Browser and device used
    o Total time spent
    o Number of time visited



    • Clicking on a user within this view displays the complete activity history of that user for the selected component.

  4. Auto Load Instead of Load More Button:
    When the Auto Load toggle is turned on, additional records will automatically load as you scroll, and the "Load More" button will not be displayed. If the toggle is turned off, users will need to click the "Load More" button to view additional records.



  5. Custom Fields

    6. Users can manage custom fields to enrich profile data. The interface provides options to Add and Delete custom fields based on organizational needs.


    Add Custom Field: You can add a new custom field by selecting a property from Google attributes, entering a display name, and choosing whether the field should be filterable by checking the provided option.


    Once added, the custom field will appear across supported views such as profile cards and list view with its associated data.


    Delete Custom Field: Any previously created custom field can be removed using the delete option provided next to each field entry.

  11. Extend custom fields search capability:
    If you want to utilize the 'starts with', 'ends with', or 'contains' search functionalities, please enable this feature. Additionally, limit its use to cases where the number of employees is under 3000.

  12. Import non M365 users:
    This feature allows you to import non-Microsoft 365 users. Before importing non-Office 365 users, please ensure to read the instructions provided below.

  13. Extend Mobile Number and Phone number search capability:
    Please activate this feature if you want to search by 'starts with', 'ends with', or 'contains', but only if the number of employees is under 3000.

  14. Restricted Access List:
    This feature limits access to certain modules or data based on user roles and permissions. Only authorized users will be able to view or modify the restricted content, ensuring data privacy and security

  15. Sync and display mobile number from azure:
    This setting will be functional when the SharePoint API is activated. For instructions on enabling it, please refer to this link.

  16. Sync employee profile information from :

    1. Graph API : The recommended configuration for the Employee Directory app is to utilize the Microsoft 365 Graph API as the primary data source to fetch all information. It is advisable to use Microsoft 365 as the data source for the majority of attributes.
    2. Sharepoint Online and Graph API: The Employee Directory app utilizes SharePoint Online and the Microsoft 365 Graph API as data sources to retrieve all information. It is recommended to enable this setting only if advised by the support or consulting team, particularly if there is a specific need such as sorting results by last name or other requirements that cannot be fulfilled using the Graph data source alone.




  17. Sync user information from

    18. In this feature, users have the option to synchronize user information from two different sources: M365 and non-M365. Alternatively, they can choose to synchronize only user information from non-M365 accounts.



  18. Topbar Filter

    19. Select Topbar filter:Users have the option to filter results by choosing either department or location.

    Default view of topbar filters: This setting allows users to select which property values they want to display as filters on the homepage. The department and location are the only two properties available for this purpose.


  19. Validate work email with SharePoint account name:
    Enable this setting to validate the work email with the SharePoint account name. Note that enabling this setting may increase the page load time of the application, so use it judiciously.
