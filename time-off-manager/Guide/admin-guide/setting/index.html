Settings

Settings page is visible only for admin role person. By default, the one installed application is an admin, you can assign admin role to person from the userâ€™s page.

  1. Office Locations:
    Here, you can add office locations by clicking on Add button. For the location we can define work week. In this table add zero for weekend days and any number on working days. Use adds more button to add multiple office location and finally click on save.



    2. Once you add office locations those location available will display in public holiday page from where you can add public holidays based on locations. These locations will show up in office location dropdown of add user panel. Public holidays and weekend leaves works for user based on location or office location

  2. Leave Type Setting:
    Here you can add new leave type by clicking on add button and entering all the details.


    1. Status: If it is enabled, it will display in apply leave form and user can apply for that leave.
    2. Accrual: Leave accrual can be enabled for as per your requirement. (In Advance setting accrual leave should be enabled and then select accrual type).
    3. Homepage: If this is enabled, it will display in home page as Leave Status.
    4. Partial Day: if this is enabled for particular leave, user can able to apply leave for partial day (0.5 or 0.25 leave).
    5. Leave Notice in Advance: You can enter the number of days you want the leaves to applied in advance.
    6. Secondary Approver:You can select HR or manager of manager or any user as seconder approver. Select HR or MoM check box and enable switch. You can type person in people picker box, click on save button and enable switch. If Fist approver (i.e., Manager of user) approve the leave request, request goes to second approver.
      Note: You can also make some second approver as auto approver from the advance settings, there you have to select the job title for second approver and cancel.
    7. Tertiary approver: You can select HR or manager of manager or any selected email id as tertiary approver

  3. Advance Settings
    From here you can add approvers, notifications to approvers, allow negative leaves, change date format, theme color, company or brand logo and icon, Graph API Integration and Auto approvals etc.
    1. Calculate leaves Include Weekends / Holidays: In case you want to include weekends as leaves, you can enable switch. Users are allowed to apply leaves for weekend
    2. HR Approvers: You can add HRs for approvers by entering their names.
    3. Additional Approvers: If you want the leaves to be approved by additional approvers, you can enable it.
    4. Remainder notifications to approvers: If any approver forgets to approve the leave, a remainder email notification will be sent if this in enabled.
    5. Allow Negative Leaves: If this is enabled, users are allowed to request the leaves even thou leaves are not available
    6. Allow Backdate Leaves: If this is enabled, users are allowed to request past can be allowed to submit back date leaves by X days
    7. Leave Accrual: It is the amount of vacation time that an employee has earned as per a company's employee benefits policy, you can enable it if you want.
    8. Allow External Domains: You can add External or guest user's domains separated by comma, e.g.: xyz.com, abc.com. You can allow guest user to request the leaves. First provide SharePoint site permission where Time off is hosted, add users in from user page from side navigation panel.
    9. Annual Leave Start Date and Annual Leave End Date: You can mention the annual leave start date and end date here. If accrued leaves are enabled, leaves will be calculated and allocated to users based on these date and date of joining of the candidate.

    4. Example

    let say
    candidate joining date = 15 th March 2021.
    Annual Start date= 1 st Jan 2021
    Annual End Date = 31st Dec 2021.
    Casual Leaves = 24.

    ✔ If accrued is disabled, candidate allocated leave would be (24 leaves /12 month) * Remaining months i.e., 9 months = 18 Leaves allocates for a years

    ✔ If accrued is enabled and type is Monthly, for march month allocated leave will be 2 and it increase by 2 on every month.

    ✔ If accrued is enabled and type is quarterly
    24 leaves/ 3 quarter for year = 8 leaves for each quarter
    Now (1st quarter= 8 leaves) / 4 months = 2 leaves
    âˆ´ 2 leaves * 2 months (remaining months to complete one quarter) = 4 leaves allocates in first quarter and then 8 leaves would allocate for in each quarter automatically.

    Allocated leaves

    1. Date Format: If you change the date format, it will update in entire application.

    2. Work Week: In this table add 0 in the cell for weekend Leave calculation will be done based on work days.

    3. Maximum Allowed Leaves to Carry Forward Next Year: You can enter the maximum number of leaves that can be carry forwarded.

    4. Enable External Users: You can enable it if you want to allow external users.

    5. Leaves Based On: Leaves are based on 3 categories: If you want to enable it just select check box and click on save button
      1. Regular: Here you can enter the total number of leaves. This leave allocates for all users based on date of joining and accrued (optional). If you want change leave for particular user, it can we done from user page.



      2. Level Based Leaves: If this is enabled, level-based leaves and grades options Will display under systems settings from where we have to first define grades for different job tile in grades then we have added leaves numbers for such grades from leave based option



      3. Tenure Based Leaves: If this is enabled, tenure-based leaves options., will display under systems settings from we you can assign leaves based on tenure


    6. Brand Logo: If you update your company/brand logo, it will update on the top of side navigation panel.

    7. Predefined Themes: You can select any of the theme color from the given colors only.

    8. Internal Data list Path: We recommend to use the data list externally so that it can connect to Power BI & Power Automate.

    9. Allow Microsoft 365 users to access app by default: By turning off, users will get access denied page if they are not added in the application. If this enable then user open the time off manager from application URL or from site content of SharePoint site, user would add in application automatically then we can edit and assign location

    10. Enable advance features - vacation in outlook, etc.: To show leaves in calendar of user and manager, we can configure graph API.

    11. Auto approval of secondary approvers: You can enter the job title of users for secondary approvers.

    12. Auto approval of tertiary approvers: You can enter the job title of users for tertiary approvers.

    13. Auto Approval Exception: You can mention leave types if you don't want to auto approval from auto approvers.

    14. Remainder notifications to approvers: Manger will get notified if there is any pending leave request of their employees

    15. Add Comp off Policy (in Days): Here number can be entered in days that user can allowed to send add comp request for worked days within specified past days

    16. Avail Comp off Policy (in Days): Here number can be entered in days so that users are allowed to take the comp off leave within specified time period else comp leave will be cancelled
