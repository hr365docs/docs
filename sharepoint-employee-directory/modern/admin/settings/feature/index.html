      Analytics Report

    The Analytics Report is available under the Advanced section. It offers two distinct views to analyze user activity:


    1. User-Based View

    This view displays user-specific data by default. It highlights the components each user has accessed, along with the total time spent on each.

    • • Clicking on a user (e.g., Isaiah Langer) displays detailed activity logs for that specific user, showing all components they have interacted with.



    • Clicking on a component within the user's record provides individual usage details for that component.


    2. Component-Based View

    This view displays a list of all components accessed by users.

    Clicking on a component reveals detailed analytics, including:

    o Users who accessed the component
    o Browser and device used
    o Total time spent
    o Number of time visited



    • Clicking on a user within this view displays the complete activity history of that user for the selected component.

      Custom fields

    The Custom Field feature allows administrators to display a variety of user information directly on the profile card and other views. This includes properties from Azure (AD properties)-(such as Extension Attributes, City, etc.), SharePoint Online (SPO) properties, and even custom fields for Non-M365 users.


    Steps to Configure:

    1. Select AD Properties: Choose the desired AD property from the available list (e.g., City, Extension Attribute, etc.).
    2. Assign a Display Name: Enter a user-friendly display name for the selected property.


    Note: To display on Profile Card: Check the Display Property Name checkbox to ensure this property appears on the user's profile card.


    To make Filterable (Optional): Check the Filterable checkbox to enable this field as a filter option next to the free search bar on the Employee Directory.


      Custom Function

      You can develop a custom function that enables users to select specific Active Directory (AD) properties and retrieve their corresponding values.


      Users can also add custom fields. For example, by selecting the Department field, users can now view records across multiple departments (e.g., IT and Sales), rather than just one.


      Only users from the selected departments — in this case, 'IT' and 'Sales' — will be displayed.


      To delete a custom function, select the one you wish to remove (e.g., a department like 'IT'), and then click the delete button.

      Exclude Options

      You can exclude individual users or groups of users using multiple filters.


      • Exclude individual users by name.
      • Exclude groups of users by domain, department, or job title.
      • Exclude users who are sign-in blocked or hidden in the GAL (Global Address List).
      • Exclude users whose names contain specific characters (minimum 3 characters).
      • Exclude shared mailbox users via the Exclude shared mailbox users option.

      Options like "Exclude users with blocked sign-in," "Exclude users hidden in address list," and "Exclude shared mailbox users" require SharePoint API access. Enable this in the advanced settings.

      To exclude users, click the appropriate dropdown or link. Select the relevant values or review the list for already excluded users.



      Exclude Users by Department

      Select the department(s) from the dropdown to exclude users based on department.


      Exclude Users by Job Title

      Select the job title(s) from the dropdown to exclude users accordingly.


      Exclude Users by Name

      Due to Microsoft query limits, you can only exclude up to 50 users by username. For more, use exclusions via department, job title, location, or name contains.


      Exclude Users by Office Location

      Image Profile Tag

      This feature allows you to apply a tag to employee profile images using a selected Extension Attribute. You can customize the tag’s appearance by selecting background and font colors.

      • Select the appropriate Extension Attribute from the dropdown menu (e.g., ExtensionAttribute1–15).
      • Choose the Background Color for the tag to enhance visual distinction.
      • Select the Font Color for text displayed on the tag.

      Once configured, the tag will be displayed as a label on the profile image in the Employee Directory.

      Restricted Access List

      The Restricted Access feature allows administrators to control the visibility of specific user properties within the Employee Directory using Include and Exclude checkboxes.

      Exclude Access — When selected, the chosen directory properties will be hidden from the specified users. This is useful for keeping certain information confidential or limited to specific groups, departments, job titles, or locations.
      Example: Salary details can be hidden from all employees except those in the HR department.

      Include Access — When selected, only the specified users will be able to view the selected directory properties. This enables tailored access, ensuring that sensitive information is visible only to a particular audience.
