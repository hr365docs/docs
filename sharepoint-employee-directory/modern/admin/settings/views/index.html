    Views

    In Employee Directory 365 you will be presented with various views. These views are structured to present you with key information.


    1. Desktop and Mobile default view: From here, you set the default view for desktop and mobile. You can select any one view (grid, list, or tile). Finally, click on the Save button.


    2. Views: From here, you set the views on the home page. It is mandatory to enable at least one view. If you enable all views, users can manually change the view. Select the views and finally, click on the Save button.


      To change the view, go to home page. Click on the icon located on extreme right top corner as shown below.


    3. Sort by: From here admin can set the Sort by based on their first name or last name or display name. It is mandatory to select at least one field.



        4. How to make users sortable on the basis of preferred name (display name) property in the Employee Directory

        Navigate to Search schema from the below URL:

        https://YourTenantName-admin.sharepoint.com/_layouts/15/searchadmin/ta_listmanagedproperties.aspx?level=tenant

        YourTenantName- Replace your tenant’s name from the above URL.

        Search PreferredName by clicking on the managed properties (on the top left side). PreferredName do not have a sortable set (Sort value is empty) as shown in the below image and hence in Employee Directory, the users with display names are not sortable. Here we have to map the PreferredName to RefinableString03 and make sorting executable in Employee Directory.


        Here we will pick an unmapped RefinableString03 property and map it to the PreferredName fields.

        Now search the refinablestring, all refinablestrings itinerary would be displayed, select and click on Refinablestring03, a search page would appear, please scroll at the bottom.



        Click on Add a mapping option as shown in the below image, crawled property selection would pop-up.


        Now search the Preferredname, select on it and finally click on the OK button as shown below.



        Once we have mapped the crawling property, Click on the OK button to finish the mapping process.


        Please wait for 8 to 24 hours and check the sorting display name in Employee Directory.

    4. User properties in grid view: From here admin can set the properties in grid view as per the requirements. Select the checkboxes of the properties in the grid view. Finally, click on the Save button. It is mandatory to select at least one property.


      That displays the properties on the grid view as shown.


    5. User properties in list view: From here admin can set the properties in list view as per the requirements. Select the checkboxes of the properties in the grid view. Finally, click on the Save button. It is mandatory to select at least one property.


      That displays the properties on the list view as shown.


    6. Use work email id instead account login id : Users details are being pulled based on UPN or work email id, default setting is off, if you want to pull records based on work email, turn it on from here.


    7. Executive(s) label :From here you can rename the executives label on the assistant’s profile card.



    8. Display profile image: The default setting is on. Admin can disable it from the toggle switch. If this is enabled, the profile images will be displayed on the home page.


    9. Clear alphabet selection with reset filter : The default setting is off. This setting will clear any alphabet selection to 'All' while clicking on reset filter icon on home page.


    10. Remove extra character from account name: The default setting is off. If it is enabled, .


    11. Show qr code icon in grid view: The default setting is off. If the admin enables this feature, it will display a QR code in the grid view.


    12. Show joyful animations: The default setting is off. If it is enabled, a celebration burst of colorful shapes on the profile card when the user has a birthday or anniversary.


    13. Show extended fields in list view : The default setting is off. If it is enabled, loading additional fields come with additional page load time, please select wisely.


    14. Show non M365 users : The default setting is off. If the admin enables this feature, it will display the non365 user's icon on the home page. You can add non-M365 users from Import under advanced settings.



    15. Show title/department/location filters as dropdown: If the admin enables this feature, it will display these properties as a dropdown on the home page.



    16. Hide mobile number for specific users:Here you can hide the mobile number for specific users in the application.


      Click on the link Hide mobile number, this displays Hide mobile number page. Enter the name and click on the Add button.
