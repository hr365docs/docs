    To set up custom-level permissions, follow these steps:
  • Open the site where the application is deployed and click on the settings icon located in the top right corner.

  • Select "Add items," "Edit items," "Delete items," and "View items." Then, scroll down and click on "Create."


  • Return to the previous page and create a group.

  • Enter the group's name and select the custom permission created in step 3. Click "create."


  • Once the group is created, add users to this group.

  • Click on "New," and enter the names of the users who will access the application. You can also select "Everyone except external users" to grant access to all users automatically, reducing manual additions and you can also add any SharePoint groups directly if needed.


  • Use the "Check permissions" feature to verify the granted permissions, ensuring only the custom permission created in step 4 is assigned.

  • Navigate to the pages, click on the "Share" button, and enter the name of the group created in step 5. Ensure that only view access is granted to the group.

    What is CORS?
    A request for a resource (like a picture) outside of the origin is known as a cross-origin request. CORS (cross-origin resource sharing) manages cross-origin requests i.e., EDM works from https://tenant-appid.sharepoint.com & if Pictures are being called from https://outlook.office.com, it treats as cross-origin resource sharing. Please click on the CORS tab to allow cross-origin resource sharing from the help in Employee Directory 365. This activity is one time until you clear cache/cookies from your browser Note: We recommend you to choose either SPO or ED (Employee Directory SharePoint library).

    How does search works in Employee Directory 365?
    Employee Directory 365 does live searches from Microsoft 365’s user database and displays information within the app at the back end, Employee Directory 365 leverages Microsoft 365’s database to provide you with live information about employees. There are two different sources where user information is stored, and Employee Directory 365 can use any one of them. One of the data sources is called SharePoint profiles where detailed information about a user is stored and another data source is Azure Active Directory where the user’s email id, free busy information, and other active directory related attributes are stored. We use Microsoft’s graph API to pull information from Azure Active Directory and Rest API to pull information SharePoint profiles.

    There are certain benefits and limitations on both date sources as some of the information, which is available in the SharePoint profile, is not available in Azure Active Directory and vice versa. Please feel free to connect with our presales team to let you know which data source would be a right fit for you. Also, you can visit advanced settings within the app to configure your choice of data source and find more info about features within the app.
    How does user sorting works in Employee Directory 365?
    Employee Directory pulls user information from Microsoft 365's user records. Graph API or SharePoint API are two methods used to pull records from Microsoft 365 based on advanced settings in the application. With the graph API employees can be sorted based on either user principle name (UPN) or display name as per "sort user by" view settings. With SharePoint API, employees can be sorted based on first name, last name, and dispaly name as per "sort user by" view settings. Custom attributes are available through Graph API hence sorting can be done using UPN or 'Display Name'.
    Why does below alert message appear and how to fix it?

    "Your search filter query has exceeded the limit of characters set by Microsoft 365. Please reduce filters/exclude users or contact your Employee Directory admin for more details".
    Microsoft has restrictions on excluding query length. To maintain the query length of excluded users, please limit this user list to 50 and use other excluded queries such as department, title, location or user contains for the remaining users.

    Navigate to the "Exclude Users List" settings, include the users designated for exclusion, and set the limit for excluded users to 50. Subsequently, you have the option to reassign the remaining users based on criteria such as location, job titles, or department, and exclude them accordingly. You may also utilize the "Exclude Users by User Contains" option to further exclude up to 50 users.

    Why am I not finding a specific user in the directory search?

    If you are unable to find a specific user in the directory, it could be due to one of the following reasons:

    The user may be excluded through one of the exclusion options configured in the application settings.

    The user might be filtered out due to custom functions applied.

    We recommend reviewing the Exclude settings, especially the conditions configured under “Exclude users contains”, to ensure the user is not being filtered out.

    If the issue persists, please contact appsupport@apps365.comfor further assistance.

    How do I update my manager, department, or job title in the directory?

    To update the manager, department, or job title, you need appropriate access to either the Microsoft Azure Portal or the SharePoint Admin Center. Changes made in either platform will automatically reflect in the Employee Directory application.

    Option 1: Update via SharePoint Online

    1. Navigate to the SharePoint Admin Center.
    2. Click on More Features.
    3. Under User Profiles, click Open.
    4. Select Manage User Profiles.
    5. Search for the user whose details need to be updated.
    6. Click on Edit My Profile.
    7. Locate the fields such as Manager, Department, or Job Title.
    8. Update the required details and save the changes.

    Once updated, the changes will be reflected in the application.





    Option 2: Update via Microsoft Azure

    1. Log in to the Azure Portal: https://portal.azure.com/
    2. In the search bar, type Users and select it.
    3. Choose the user whose details you want to update.
    4. Locate the fields for Manager, Department, or Job Title.
    5. Update the required information and save the changes.

    Once updated, the changes will be reflected in the employee profile within the application.

    How can I downgrade to a previous version of the Employee Directory 365 application?

    Follow these steps to safely revert to an earlier version of the Employee Directory 365 application:

    • Access the AppCatalog directly via the following link:
      https://yourtenantname.sharepoint.com/sites/appcatalog/_layouts/15/tenantAppCatalog.aspx/manageApps

    • Navigate to the AppCatalog and search for the Employee Directory 365 application.


    • Select the application and click on More Options.


    • Click on Version History.


    • Select the version you wish to restore, then choose the appropriate time and date.


    • Options such as View, Restore, and Delete will be displayed.


    • Click Restore and confirm the action by selecting OK.


    • Once restored, verify the version number displayed below the selected time and date.


    • After confirmation, select the package, click on Files, and choose Deploy.


    • In the deployment popup, check the required checkbox and click Deploy to finalize the process.
