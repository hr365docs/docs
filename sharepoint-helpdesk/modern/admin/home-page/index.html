    Home Page

    The Home Page in Helpdesk 365 serves as the central hub for managing and tracking tickets. It offers multiple ways to view, filter, and interact with tickets, allowing users, agents, and team leads to monitor ongoing work efficiently. The interface includes top-level ticket categories, powerful search capabilities, and multiple view modes to streamline ticket management.




    Ticket Navigation Tabs

    At the top of the Home Page, users can quickly navigate between key categories of tickets. These tabs display real-time ticket counts and act as filters to instantly update the view.

  • Unassigned – Displays all tickets that have not yet been assigned to any team or agent.


  • Team Tickets – Shows tickets assigned to your team, enabling team leads to monitor workloads.


  • My Tickets – Lists tickets assigned specifically to the logged-in agent for quick task tracking.


  • My Requests – Shows tickets raised by the logged-in user, helping them track progress and updates.


    • The Ticket Status Tabs allow users to quickly switch between tickets based on their current progress stage. These tabs act as instant filters that update the ticket list in real time, enabling faster navigation and efficient ticket monitoring.

  • Open – Displays all active tickets that are currently in progress or awaiting action. These tickets require attention from the support team.


  • Resolved – Shows tickets that have been marked as resolved by the support team. The issue has been addressed, and the ticket awaits requester confirmation or closure.


  • Closed – Displays tickets that have been fully completed and officially closed. No further action is required on these tickets.



    • Custom Views

    The Custom Views feature allows users to create personalized ticket layouts based on their preferred columns, filters, and view types. These views help streamline workflows by displaying only the most relevant ticket information.

    1. Open the Select View dropdown from the Home Page.

    2. Select Configure View to open the Custom View panel.


    3. Click the + Add button to create a new custom view.


    Create & Customize a View

    Users can customize their view by selecting the desired options:

    • Custom View Title – Name your view for easy identification.
    • View Type – Choose between Modern View or Classic View.
    • Add Filters – Apply filters such as Priority, Status, Teams, Services, etc.
    • Select Tabs – Choose which tabs (Unassigned, Teams Tickets, My Tickets, My Requests) should appear in the view.
    • Customize Columns – Arrange and select columns for each chosen tab.

    After configuring the layout and filters, click the Save button to store the custom view. The saved view will now appear in the View dropdown list for quick access.


    Share Custom Views

    Users can share their custom-created views with specific team members using the Share option.



    Ticket Creation

    Users can create new tickets directly from the Home Page using the + Ticket button. This allows them to report issues, request services, or raise inquiries quickly and efficiently.

    1. Click the + New Ticket button to open the ticket creation form.

    2. Enter the required details such as Title, Teams, Services, Sub Services, Priority, Request Type, and a clear Description of the issue.

    3. If enabled, the system can automatically assign Teams based on selected Services and Sub Services.

    4. Users may also add CC recipients and choose the preferred communication method if applicable.



    1. Users can attach supporting documents, screenshots, or files using the drag-and-drop feature. Simply drag files into the attachment area or use the upload option provided in the ticket form.


    1. After reviewing the details, click Submit to create the ticket.


    Search and Filters

    The Global Search allows users to quickly locate tickets across the entire Helpdesk 365 system. To use this feature, first select a search category such as Ticket ID, Assigned To, or Requested By. Once a category is selected, the system will search through all matching records and display the results instantly based on the selected field.


    The date range dropdown allows users to quickly filter tickets based on predefined time periods. By selecting an option such as Current Month, Current Quarter, Last Month, Last Two Months, or Last Three Months, the ticket list automatically updates to display only the records created within the chosen timeframe. This helps users easily review and analyze tickets for specific reporting periods without manually entering dates.



    Search Bar: Users can click the search icon to open a search field where they can instantly look up tickets by entering keywords such as Ticket ID, Title, Status, Priority, or any related text. As the user types, the ticket list dynamically updates to display only the matching results.



    Advanced Filter Panel: By selecting the filter icon, users can apply detailed filters including date range, teams, services, sub-services, request type, priority, status, and assigned agent. After choosing the desired criteria, users can click Apply Filter(s) to narrow the results or use Clear Filter(s) to reset all selections.


    Sorting Tickets

    Users can sort tickets based on fields like Teams, Services, Sub Services, Priority, Request Type, and more.

    Advanced sorting options allow ordering results in ascending or descending order.



    View Modes

    The Helpdesk 365 Home Page provides three different viewing modes—Classic View, Kanban View, and Modern View. Each mode offers a unique way to visualize and manage tickets, allowing users to choose the layout that best suits their workflow. These views can be switched anytime from the view selection menu.

    1. Classic View
    2. Kanban View
    3. Modern View

    Classic View

    Classic View displays tickets in a structured table format, making it easy to compare and scan ticket details at a glance. It includes sortable columns such as Ticket ID, Title, Status, Priority, Services, Sub Services, Assigned To, and Created Date. Users can sort any column in ascending or descending order for quick analysis.

    • Provides a familiar spreadsheet-style layout for clear ticket overview.
    • Supports column sorting and filtering directly within the table.
    • Ideal for agents who prefer data-intensive views and quick sorting.
    • Shows all tickets in rows, enabling easy navigation across large datasets.



    Kanban View

    Kanban View presents tickets as movable cards organized into columns such as Unassigned, Teams Tickets, Teams Closed, and My Tickets. This visual workflow helps users manage tickets more intuitively and prioritize work based on status.

    • Drag and drop tickets between columns to update their status instantly.
    • Each ticket card displays essential details like Title, Priority, Created Date, Team, and Assigned Member.
    • Offers filters at the top (Teams, Members, Status, Priority, Request Type) to narrow ticket visibility.
    • Enhances productivity by giving a board-style overview of ongoing tasks.



    Modern View

    Modern View displays tickets in a clean, card-based layout, offering a more readable and detailed presentation compared to Classic View. Each ticket card highlights the title, requester details, request type, priority, description, services, and created date.

    • Shows ticket information in compact cards for a visually appealing experience.
    • Includes inline status and priority update options for quick modifications.
    • Ideal for users who prefer descriptive ticket previews rather than table rows.
    • Provides faster navigation with improved spacing and readability.



    The Sync button refreshes the ticket list and reloads the most recent data from the Helpdesk 365 system. It ensures that users always view the latest ticket updates, status changes, and newly created records without needing to reload the entire page.


    Notification

    The Notification feature alerts users about important updates related to their tickets. When new comments, status changes, assignments, or actions occur, a notification icon displays an indicator to highlight unread alerts.


    Assign and Update Tickets

    Users can assign or update ticket status directly from the Home Page.



    Bulk Assign Tickets

    Multiple unassigned tickets can be selected at once and assigned in bulk.



    Close the Ticket on Behalf

    Agents can close a ticket on behalf of the requester when needed by selecting the appropriate option.
