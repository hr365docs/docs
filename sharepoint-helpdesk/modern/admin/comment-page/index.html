    Comment Page

      Some "ticket actions" are shown below on the comment page :

    1. Reply
    2. Private Notes
    3. Consult
    4. Transfer
    5. Merge
    6. Split
    7. Escalate
    8. Review
    9. Sub Tickets
    10. Requester Detail
    11. Scheduled Reply
    12. Edit and Download

    When you click on a Ticket ID or Title from the home page, you are redirected to its respective comment page.

      From this page, you can update ticket properties such as status, assigned team, team member, and more.


        Reply

        1. Use the Reply option to respond directly to the user’s message.


        2. The agent’s message is entered in the message editor, where formatting options are also available.


        3. Once the reply is submitted, the message is sent to the requester.



        Private Notes

        1. You can add Private Notes for internal agent communication. These notes are not visible to the requester.


        2. Private notes generate an internal email notification to selected team members when added.


        3. These private messages remain hidden from the requester.


        4. Ticket assignees receive notifications when a private comment is added by team members.


        Consult

        1. The Consult option allows you to communicate with team members before transferring a ticket.


        2. You may discuss ticket allocation or request assistance. Clicking the Consult button sends an email notification to the selected team member.



        Transfer

        1. The Transfer option allows you to move a ticket from one department to another. You may assign the ticket to a specific team member and/or department.


        2. When a ticket is transferred, the assigned team or agent receives an email notification.


        Merge Tickets

        When the Merge Tickets feature is enabled by the admin, the option becomes available on the ticket comment page. Admin or the ticket assignee can merge the current ticket into another ticket using its Ticket ID.

        Once tickets are merged, the old ticket ID is closed and linked to the new one. A merge notification is automatically sent to the requester.


        To merge tickets, follow the steps:

        1. Open the ticket by clicking the Ticket ID from the home page.

        2. On the comment page, click the Merge button to open the merge confirmation popup.


        3. Click Yes to continue.


        4. Enter the Ticket ID you want to merge into.

        5. Click the Merge button to complete the process.



        Split Tickets

        Split Tickets is used when a single ticket involves multiple departments or requires separate tracking. Admin must enable this feature from settings.


        To split a ticket, follow these steps:

        1. Open the comment page by clicking the Ticket ID from the home page.

        2. On the comment page, click the Split button to open the split confirmation popup.



        3. Click Yes to proceed.


        4. Enter the New Ticket Title, add your comments, and select the required Teams and Services for the new ticket.

        5. Click the Split button to create the new ticket.


        Escalate Tickets

        The Escalate Tickets feature allows agents to escalate a ticket when an issue remains unresolved within a specific time period. Escalation notifies higher-level teams or supervisors so that appropriate action can be taken. This feature must be enabled by the admin in the settings.


        To escalate a ticket, follow the steps:

        1. Go to the home page and click on the Ticket ID to open the comment page.

        2. Click the Escalate button. A comment box will appear where you can enter escalation remarks.


        3. From the Add/Edit Teams section in settings, you can create new escalation teams using the New EscTeams option.


        4. After selecting a team, you can configure escalation details such as Level (L1/L2), Supervisor, Email ID, and Team Members in the Add Escalate Teams panel.


        5. On the ticket comment page, the selected Escalation Team and the corresponding Team Members will appear in dropdowns. Once selected, click the Escalate button to finalize.


        6. Finally, click the Escalate button to submit the escalation.



        Review Tickets

        The Review Tickets feature allows supervisors or designated team members to evaluate and review ticket activity, quality of work, and overall handling. This feature must be enabled by the admin from the settings.


        To review a ticket, follow the steps:

        1. Open the ticket comment page by clicking the Ticket ID on the home page.


        2. Click the Review button. A comment box will appear where you can enter feedback or review notes.

        3. Select the team members you want to share the review with.

        4. Submit the review using the Review button.


        5. After submitting the review, the assigned agent will receive a notification prompting them to review the ticket.


        Sub Tickets

        Sub tickets allow agents to divide a main ticket into multiple smaller tasks that can be assigned to different teams or team members. This feature is disabled by default and can be enabled by toggling the Enable Sub Ticket switch.

        Once enabled, the Sub Ticket option appears on the comment page.


        To create a Sub Ticket:

        You can create multiple sub tickets from a single parent ticket.

        1. Go to the ticket comment page and click the Sub Ticket option.

        2. Select Yes on the confirmation popup to proceed.

        3. Enter the Sub Ticket Title and add Comments.

        4. The sub ticket will be created and displayed on the home page.

        Requester Details

        1. In the contact info section, you can click the Calendar icon to create a meeting with the requester.


        2. Add the user(s) you want to invite to the meeting.


        3. Optionally, add additional participants similar to adding CC recipients.


        4. You can also invite Non-M365 Users by entering their email address.


        5. After adding participants, select the date and time, add a description, and enable the Teams Meeting toggle. Click Send to issue the meeting invitation via email.


        6. Use the MS Teams Call and MS Teams Chat icons to initiate a call or chat directly with the requester.


        7. Admins can update or change the ticket requester's name if required.


        Scheduled Reply

        The Scheduled Reply feature enables users to compose a message and schedule it to be delivered automatically at a specific date and time. This is helpful for planned responses or sending updates outside working hours.

        When the timer icon is clicked, the Send Scheduled Message popup appears.


        The popup contains a Date Picker and a Time Dropdown (HH:MM AM/PM format). Only valid future time slots are displayed.


        Choose the schedule time and click Send to confirm the scheduled reply.



        Once scheduled, the system will send the message automatically at the selected time.

        Scheduled message delivery is powered by Power Automate, which ensures messages are sent at the specified schedule.

        Edit and Download

        The Edit and Download options allow users to conveniently modify ticket information and export a complete copy of the ticket for offline use or record-keeping.

        Clicking the Edit icon lets you update the ticket title quickly and easily.


        The Download option allows you to export the ticket as a PDF file.


        When you click Download, the system generates a PDF containing the complete ticket details — including the description, comments, attachments, and activity logs.
