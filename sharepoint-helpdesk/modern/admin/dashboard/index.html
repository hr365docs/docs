    Dashboard

    The dashboard provides a comprehensive visual overview of ticket activity, SLA performance, and operational trends. Each section of the dashboard displays key metrics that help users quickly understand workload distribution and service efficiency.


    Ticket & SLA Charts

    The first set of charts provides a high-level overview of ticket volume and SLA performance.


    Tickets This Month — Shows the total number of tickets logged throughout the current month and can be filtered by team.

    SLA Response – Average — Displays the overall average SLA response time for the selected date range to track SLA compliance.

    SLA Response – Weekdays — Breaks down SLA response times by weekday to help identify performance trends across different days.


    Ticket Distribution Charts

    This section helps users understand how tickets are categorized and distributed across different operational groups.


    Tickets by Type — Donut chart showing ticket counts by request category (e.g., hardware, software, access), used to identify common request patterns.

    Tickets by Teams — Displays how tickets are distributed across departments/teams to evaluate resource utilization and workload balance.

    Tickets by Priority — Highlights the number of tickets by priority level (Low, Medium, High), providing insight into urgency trends.


    Performance & Feedback Metrics

    This section reflects customer experience, support quality, and ticket origination trends.


    Customer Satisfaction — Shows the satisfaction percentage based on survey responses; can be filtered by team for detailed evaluation.

    Knowledgebase (KB) Effectiveness — Indicates how effective KB articles are in resolving issues based on user engagement and feedback.

    Tickets by Source — Shows where tickets originated (Portal, Email, Forms), helping understand user behavior and preferred submission channels.


    Operational Performance Charts

    The final section includes detailed charts that provide insights into operational efficiency and SLA resolution trends.


    Time Track — Line chart showing time spent on resolving tickets during the selected month, helping identify workload and throughput patterns.

    SLA Resolve – Weekdays — Displays the average resolution time for each weekday to identify daily variations in performance.

    SLA Resolve – Average — Provides the overall average time taken to resolve tickets within the selected period to monitor SLA adherence and efficiency.

    Status-Based Dashboard View

    Dashboard charts can now be filtered by clicking on any ticket status, and the dashboard results will update automatically based on the selected status.



    Services Scroll View

    Enhanced the dashboard's visual presentation to effectively display up to 50 services, ensuring a clean layout, improved readability, and seamless user experience. Users can now view services in a scrollable format with the total number of services displayed.


    View Options

    The icon highlighted in red represents a "View Options" feature. By clicking this icon, users can access a dropdown menu with two options: Graphical View, Tabular View


    When the user selects Graphical View from the dropdown, the data is displayed in a visual chart format. This allows users to quickly analyze and understand the data through intuitive visual representation.


    When the user selects Tabular View from the dropdown, the data is displayed in a visual table format. This allows users to quickly analyze and understand the data through intuitive visual representation.


    Global Dashboard Filter

    User can filter the Dashboard data globally with the help of this globally filter icon



    Select the dropdown option and click on Apply Filter button.



    Dashboard Settings


    From here, you can customize which charts you want to display on the dashboard for each row.

    1. Click the Configure button next to Dashboard Settings.


    Show SLA Charts

    In Show SLA Charts, there are two options. The first is Based on Month. After selecting it the data in the dashboard will be displayed according to the selected month.


    Here, the user can view SLA data for any month of their choice, such as the current month, last month, or any other month.


    The second option is 'Based on Teams,' which allows the user to select and view SLA data categorized by teams as per their preference.


    In this case, the data will now be displayed based on Teams, whether it’s all teams or specific other teams.


    Dashboard Rows Setup

    1. Dashboard Settings panel will open. There are four tabs: Row 1, Row 2, Row 3, and Row 4.

    2. Select the checkbox option of charts which you want to show in the dashboard row.

    Top Header

    In the Top Headers section, you can customize which ticket statuses appear at the top of your dashboard. Select the statuses you want to highlight, and then click the Save button to apply your changes.


    These customized headers help you quickly identify and filter tickets based on the selected status in the dashboard charts.
