    Reports

    The Reports module allows Admins and Supervisors to view, analyze, and download ticket-related data. Users can filter reports based on date range, assigned agent, teams, status, priority, request type, and services. Once the desired filters are applied, the data can be exported to Excel for further analysis.


    1. Teams Report
    2. Agent Report
    3. Requester Report
    4. CSAT Report
    5. First Contact Resolution (FCR)
    6. Global Filter Behavior

    Teams Reports

    The Teams Report provides ticket information grouped by departments/teams. Users can filter by team, service, priority, assigned agent, request type, and date range. This report helps in analyzing team workload, performance trends, and service categories.


    When a user clicks on any team name in the Teams Report, a detailed page opens displaying ticket insights specific to that team. This page includes two tabs: Agent Reports and Requester Reports.


    Agent Report

    The Agent Report provides detailed insights into tickets handled by individual agents. Admins see all agents, while supervisors only see agents from their assigned teams. The report displays key metrics such as:


    The Agent Tickets page provides complete visibility into every ticket managed by the agent, including ticket ID, title, request type, status, priority, services, department, and sub-services. This helps supervisors verify ticket workload, resolution progress, response efficiency, and SLA adherence for each agent.


    Requester Report

    The Requester Report provides a detailed breakdown of tickets raised by each requester. Admins can view all requesters across all teams, while supervisors only see requesters belonging to their assigned teams. This report helps in understanding requester trends, issue frequency, and overall service demand.


    When the user clicks on a requester name, the system opens the Requester Tickets page. This page displays every ticket created by that requester, including ticket ID, title, number of calls, request type, status, priority, services, department, and sub-services.




    CSAT Report (Customer Satisfaction)

    The CSAT Report displays customer satisfaction feedback based on survey responses. Users can filter results by team, service, ticket owner, and date range.

    Survey feedback shows three rating categories: Positive, Neutral, Negative.


    First Contact Resolution (FCR)

    The First Contact Resolution (FCR) report tracks how effectively tickets are resolved during the first interaction. It helps understand agent performance and customer service quality.


    Single Comment – Auto FCR

    If an agent resolves or closes a ticket after adding only one comment, the system automatically marks the ticket as FCR = Yes.



    Multiple Comments – Confirmation Required

    If the agent adds between 2 to 5 comments before resolving the ticket, a pop-up asks: “Was this ticket resolved on first contact?” Agent selection determines the FCR value:

    • Yes → FCR = Yes
    • No → FCR = No


    More Than 6 Comments – Auto Non-FCR

    If an agent adds six or more comments before resolution, the system automatically marks the ticket as FCR = No. This ensures the FCR metric reflects true one-interaction resolutions.


    Global Filter Behavior

    When the user applies a Global Filter such as Current Month, Last Month, or other predefined ranges, the system restricts the data strictly to that selected timeframe. During this period, other filters like agent, team, priority, and status become inactive to prevent conflicting filter combinations. To use other filters again, the user must first reset/remove the global filter.


    Total Ticket Count & Search Filters

    The report dynamically displays the total number of tickets based on the applied filters. Examples include filtering by Priority or Status such as “High Priority” or “Resolved.”



    Access Permissions : The visibility of reports depends on the user role:

    Admin – Can view all tickets across all teams.
    Supervisor – Can view only the tickets belonging to their assigned teams.
    Agent – Can view only their own ticket data.
