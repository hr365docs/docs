    What are the different types of roles in helpdesk application and what are their importance?
    There are 4 types of roles namely admin, supervisor, and agent.

    Admin:
  • Has full access.

    • Supervisor:
  • Able to assign all teams tickets to their agent if he/she has global or team Ⅰ ticket scope access.
  • Able to assign their teams tickets to their agent if he/she has global, team ticket Ⅰ or Ⅱ scope access.
  • Able to change the status of other team’s tickets if he/she has global scope access only.
  • Able to delete their team tickets.
  • Able to generate reports of their teams’ tickets.
  • Can’t change the status of their own raised tickets.

    • Agents
  • Able to assign all teams tickets to their agent if he/she has global or team access Ⅰ ticket scope.
  • Able to assign their teams tickets to their agent if he/she has global, teams’ access Ⅰ or Ⅱ scope.
  • Able to change the status of other team’s tickets if he/she has global access only.

    • User
  • Can only raise the tickets.
    • Note: By default, all users have user role access.
    What are the different types of tickets scope?
    Global access
  • Can view all teams’ unassigned tickets in the Helpdesk.
  • Can assign tickets for all teams.
  • Can edit or update a ticket of all teams from the comment page only.

    • Team access I
  • Can view all teams’ unassigned tickets in the Helpdesk.
  • Can assign tickets for all teams.

    • Team access II
  • Can view all teams’ unassigned tickets.
  • Can assign tickets for their teams.

    • Restricted access
  • Can only view their teams’ assigned tickets.
  • Team tickets detailed view will not be available.
  • Can update only tickets assigned to him/her.

    Why is the "New Ticket Created" notification sent multiple times?

    The "New Ticket Created" notification is sent based on the roles assigned to the user initiating the action. If an administrator also holds agent and supervisor roles, the application processes the ticket creation independently for each role.

  • Notifications are triggered per role-specific flow when a ticket is created.
  • Admins assigned as agent and supervisor will receive the notification multiple times—once for each role.
  • This ensures each role-based function is acknowledged within the ticketing workflow.

    • To reduce redundancy in such notifications, it is advisable to review role configurations and tailor them to suit organizational needs.

    Why does the Helpdesk application's performance degrade?

    The Helpdesk system's performance may degrade due to extensive error logging enabled by default. Continuous logging activity consumes system resources, resulting in slower response times and reduced efficiency.

  • Error logs can accumulate rapidly, especially in high-traffic environments.
  • Logging operations may interfere with real-time processing and UI responsiveness.

    • To mitigate this issue, it is recommended to disable the error logging toggle available under General Settings. Doing so can significantly improve application performance and ensure a smoother user experience.


    How to Reconfigure the Shared Mailbox ID in SharePoint Power Automate Workflows
    Procedure to Update the Shared Mailbox Identifier in SharePoint Power Automate :

    1. Navigate to SharePoint Power Automate.


    2. Click on Power Automate in the top menu and select Solutions.


    3. Open the SharePoint Admin Centre.


    4. In the left navigation, click on Teams & Groups, then select Shared Mailboxes.


    5. Choose the appropriate shared mailbox.


    6. Make sure that the shared mailbox has the required permissions assigned: Read, Send As, and Full Access.



    7. In Power Automate, under Solutions, select Help Desk 365.


    8. Click on Environment Variables.


    9. Locate the Support Mailbox ID variable and click to open it.


    10. Update both the Default Value and Current Value with the new shared mailbox ID.


    11. Click on Cloud Flows to view your automated flows. Locate HR365-HD Shared Mailbox, click the vertical ellipsis icon, and select Edit.


    12. In the flow editor, click on the trigger When a new email arrives. Then, click the folder icon and choose the appropriate inbox folder.


    13. Once the folder is selected, click the Save icon to finalize the update.


    What is the difference between the "Created By" and "Requested By" columns?
    1. When a ticket is created directly through the application, both the Created By and Requested By fields display the same information — the user who initiated the request.
    2. However, when a ticket is generated via Power Automate, the Created By field reflects the account credentials under which the flow connection is configured, as all automated actions run under that account. In this case, the Requested By field identifies the actual requester of the ticket.

    Note: If the Created By column creates confusion in reporting or visibility, you may adjust the view by navigating to Column Settings and removing this column, since the Requested By field should be considered the authoritative source for identifying request ownership.

    What file types are allowed as attachments when creating a ticket from email?

    The Helpdesk application supports all file formats as attachments when creating an email ticket, except for .msg and .oft files. These two formats, which are commonly used for email attachments, are restricted due to Microsoft’s security limitations. Additionally, Power Automate will block attachments of these file types, ensuring they cannot be processed through the system.

    How can I downgrade to a previous version of the Helpdesk application?

    Follow these steps to safely revert to an earlier version of the Helpdesk application:

    • Access the AppCatalog directly via the following link:
      https://yourtenantname.sharepoint.com/sites/appcatalog/_layouts/15/tenantAppCatalog.aspx/manageApps

    • Navigate to the AppCatalog and search for the Helpdesk application.


    • Select the application and click on More Options.


    • Click on Version History.


    • Select the version you wish to restore, then choose the appropriate time and date.


    • Options such as View, Restore, and Delete will be displayed.


    • Click Restore and confirm the action by selecting OK.


    • Once restored, verify the version number displayed below the selected time and date.


    • After confirmation, select the package, click on Files, and choose Deploy.


    • In the deployment popup, check the required checkbox and click Deploy to finalize the process.
