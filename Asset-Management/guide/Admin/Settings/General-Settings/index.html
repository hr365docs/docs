General Settings


    Here you can set date format, currency, asset accept notification, system generated barcode, theme color, brand logo etc.


    1. Currency: From here, you can select the type of currency from the dropdown. Click on the Save button.


    2. Date Format: From here, you can select the date format from the dropdown. Click on the Save button.


    3. Asset ID Prefix: Asset ID Prefix specifies the letter prefix for the generated Asset ID.


    4. First Asset ID: The asset ID is the unique number that will start after you have entered it. Make sure that you must enter the number with at least 3 digits.


    5. Maintenance Module Title Text:


    6. Depreciation Method: You can enable depreciation feature, just selecting either Straight Line or Diminishing Balance method as per your requirement and finally click on Save button.


    7. System Generated Barcode: If this is enabled, barcode generate automatically on adding assets.


    8. External User: If you enabled it, you could add external users or guest users in the application and can assign assets to them.

      Make sure guests or external users SharePoint site permissions should be provided where the Asset management application is hosted.


    9. Internal Datalist Path: If this is enabled, App data can b stored in SharePoint list within the app or externally at the site/subsite collection level where app is installed.


    10. Show projects while assigning assets: If this is enabled, project option will display while assigning assets in pop, you can assign assets to the users along with project.


    11. Camera Scanner: If this is enabled, camera scanner tab will display in home page from where you can scan barcode of the assets to search the assets.


    12. Retain Asset's Last Location: If this feature is enabled, the user’s location will retain on available assets after returning the assets. You can see retained location in user location field of the assets.


    13. Assets accept notification: If it is enabled, when asset is assigned to users, they will receive email notification from where use can accept/reject assets. If it is disable when asset assigned to users, email notification we not send and those assets will show in acknowledgement status page later we can initiate.


    14. Assign Assets to Location: If this is enabled, location option will display while assigning assets in pop, from where you can assign assets to the location.


    15. Assign Assets to Non M365 Users: If this is enabled, assign assets to non M365 users’ option will display while assigning assets in pop, from where you can assign assets to non O365.


    16. Assets Advance Booking: if you disabled this asset booking feature disable from the application.


    17. Enable Software: If this this is enabled, software tabs in home page will displays, one tab is used to sync O365 license another is used to add software manually and can assign to users.


    18. Asset Location Mandatory: If this this is enabled, you can get the asset location.


    19. CC to Location Email Id: If this this is enabled, email notification will be sent to location.


    20. Allow Asset Manager to Add Assets: If this this is enabled, Manager can add Assets.


    21. Allow Asset Manager to Delete Assets: If this this is enabled, Manager can delete Assets.


    22. Assign & Return Assets Option in Side Navigation Panel: You can hide assigned and return assets in navigation panel just disabling it.


    23. Allow User to Book Assets: If this this is enabled, Users can Book Assets.


    24. Select Approvar to Book Assets: If this this is enabled, Users can select approver.


    25. Availability of Assets During Maintenance: If this this is enabled, Users can book asset whcih is in maintenance.


    26. Maintenance Module: If this this is enabled, you can add Maintenance Frequency and Maintenance Due Date while adding assets.


    27. Hide Asset ID Prefix: If this this is enabled, You can hide Asset ID prefix.


    28. Consumable Module: If this this is enabled, consumable tabs will displays on the home page.


    29. Allow Users to Request Consumable: If this this is enabled, users can raise request of consumable item.


    30. Allow Shipping of Consumable: If this this is enabled, users can request for shipping of consumable item.


    31. Consumable Module Title Text:From here you can rename module title.


    32. Serial Number Mandatory: If this this is enabled, entry of serial number is mandatory.


    33. Asset Acknowledgement: If you type acknowledgement text, user can see in acknowledgement page (acknowledgement page link is available in assigned email notification of the user).


      Create PDF of asset acknowledgement: When you enable this and setup Power Automate, pdf file will create and stored in SP library and sent to user who has acknowledged the assets.

      34. Below mentioned is the URL where you can find pdf files

      https://YourTenantName.sharepoint.com/sites/SiteName/HR365AMPAssetdoc

    34. Barcode Customization: From here you can customize barcodr by adding Barcode Title Text, Organization Name, Asset Field Attribute and enabling Set Asset Field Attribute for Barcode.


    35. SCCM Integration: Refer to SCCM Integration


    36. Software & MS Intune Integration: If you integrate it, Intune assets will sync in the application.


    37. Enable Theme Color: You can select color as per your requirement and click on Save button.


    38. Brand Logo: You can upload logo by clicking on Upload logo. You can crop the logo and finally click on Save Logo button.
