  • Installation of applications from Outlook

    Prerequisites:

    1. The application can only be installed by a Global Admin or SharePoint Admin by following the steps outlined below.
    2. The application’s features rely on APIs, which can be enabled by a Global Admin with the necessary permissions.

    1. Open your browser and navigate to your SharePoint tenant site.
      Example: https://clayfly.sharepoint.com/#/

    2. Click on the App launcher (grid icon) at the top-left corner of the page.

    3. From the list of Microsoft 365 apps, scroll down if needed and click on the Admin icon (highlighted with a gear and 'A').



    4. After clicking on the Admin icon, you will be redirected to the Microsoft 365 Admin Center.

    5. In the left-hand navigation panel: Click on Settings to expand its options. Then click on Integrated apps from the submenu.


    6. On the Integrated apps page, you will see a list of deployed apps for your tenant users.
      To add a new custom app, click on the Upload custom apps button located near the top of the page.


    7. On the Upload Apps to deploy page:

      • • The App type is automatically set to Office Add-in (grayed out).
      • • Select Upload manifest file (.xml) from device.
      • • Click Choose File and browse to select the manifest XML file (e.g., manifest 9.xml).


    8. Select users to assign the app (Just me, Entire organization, or Specific users/groups) and click Next.


    9. Review the app permissions and capabilities, then click Next to continue with the deployment.


    10. Review the deployment summary and click Finish deployment to complete the app deployment process.


    11. The app Helpdesk365(Outlook) has been successfully deployed and is now listed under Integrated apps in the Microsoft 365 admin center with a status of OK for Outlook.


    12. Click on the "New mail" button


    13. Then, from the dropdown, click on Mail to open a new mail compose window.


    14. After opening the new mail window, go to the Insert tab at the top.
      Click on Apps (or Get Add-ins) to access the installed Outlook apps, including your Helpdesk365 - Contract Management app.



    15. In the Get Add-ins window, use the search box at the top right.
      Type Helpdesk in the search box to quickly find the Xdesk 365.


    16. Then, select or add the app from the search results to use it in your mail.


    17. You will see Helpdesk listed under Apps.
      Click on Helpdesk to use the its features directly within your Outlook mail.


    18. When you click on the Helpdesk app, select Show Task Pane to open it.


    19. Next, enter the app’s site URL and then click Submit.


    20. As shown in the screenshot below, you will need to enter a site URL in a similar format.


    21. Click the Submit button to proceed.


    22. You must sign in to the app using your Microsoft account.


      23. After successfully signing in, users will gain access to the features and options available in Helpdesk365, as shown below.


      1. Create a New Request:
      When the user selects "Create New Request," they can initiate a new request. Additionally, users have the option to generate a request using AI within the same flow. To use this feature, AI integration must be enabled in the application settings.


      i. Create:
      When the user clicks "Create," all the necessary fields for submitting a request will be displayed.


      Once all the required fields are completed, the user can submit the request by clicking the Submit icon or save it as a draft by clicking the Editable Draft icon.


      Once all the necessary fields are completed, the user can submit the request by clicking the Submit icon or save it as a draft by clicking the Editable Draft icon


      Your request has been submitted successfully.


      ii. AI Request: AI Request provides an additional feature that allows users to fetch data directly from the selected file.


      Submitting a Request Using Outlook Attachments

      Users must select a file from the available options in the Outlook attachments, as shown below.

      Once a file is selected, the system will automatically extract the available field information from the attachment.

      After the fields are populated, review the details and click Submit to create the request.


      iii. Update an Existing Request

      When the user clicks on "Update Existing Request," a list of previously created requests will be displayed.

      The user can then select the appropriate request they wish to update from the list.


      The user can select a request to update, modify the necessary fields, and then click Update and Submit to save the changes.


      Your request has been updated successfully.


      iv. Retrieve Files from Contract Management

      Users can retrieve attachments related to a specific request by selecting this option.

      Once the user clicks the “+” icon, the selected attachments will appear in the box on the left side.

      A list of all existing requests will be displayed for the user to choose from in order to access the corresponding attachments.


      v. Adhoc Tasks

      The Adhoc Tasks option displays a list of all tasks from the application along with their relevant details.

      There are two types of installations:

      1. Installation via package upload in the app catalog site.
      2. Installation from the SharePoint store (AppSource).

      If you delete the package installation and reinstall the app from the SharePoint store, or vice versa, it will not impact or result in data loss. This is because the application's data is stored safely in SharePoint lists, which remain unaffected by the installation method.
