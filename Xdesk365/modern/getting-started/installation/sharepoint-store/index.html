Installation of Xdesk 365 from SharePoint Store

Prerequisites:

  1. The application can only be installed by a Global Admin or SharePoint Admin by following the steps outlined below.
  2. The application’s features rely on APIs, which can be enabled by a Global Admin with the necessary permissions.

  1. Navigate to the app catalog page using the following URL format:
    https://yourtenantname.sharepoint.com/sites/appcatalog/_layouts/15/tenantAppCatalog.aspx/manageApps

    Replace yourtenantname.sharepoint.com with your actual tenant name, for example: cubiclogics.sharepoint.com.
    Once you are on the page, click on the "Classic Experience" options.



  2. Click on setting gear icon and select on Add an app.



  3. Click on SharePoint Store and type Apps365 in search box. All applications from Apps365 will sort.



  4. Click on Xdesk 365 application.


  5. Click on Add to Apps site.


  6. Select the checkbox (Enable this app and add it to all sites ) and click on Add button.

    7. Now you can make this solution or application package available for all sites in the organization, once the application is installed, later you can remove the app from all sites.

    By performing these actions, the selected application package will be downloaded from the Microsoft SharePoint Store into your tenant.


  7. Click on OK button.


  8. On SharePoint admin center page,From left panel under advanced click on API access option under pending request, there are requests ( Microsoft Graph, Presence.Read.All, User.Read.All, MailboxSettings.Read, Calendars.Read, Group.Read.All, Chat.ReadWrite, Chat.Create, Mail.Read.Shared) select and click on approve one by one for all the requests.



  9. The user can select and approve individual permissions, then proceed further.




  10. Go to Location (i.e., Site collection or subsite) where you want to install the Xdesk 365 application.

    11. We have to add Xdesk 365 as the web part on the page.


  11. Now click on + icon and then select the Helpdesk 365 as shown in below.



  12. For a cleaner appearance, you can also change the title areas to plain by clicking the Properties icon and selecting Plain, as shown below.


  13. The Quick Setup in Xdesk 365 allows users to configure their helpdesk efficiently. The setup includes the following key steps:


    14. Add team members to Xdesk 365 to enhance collaboration and streamline tasks.




    When a team is successfully added in Xdesk 365, the status changes to "Completed", indicating that the task is finished.



    Define services to categorize issues and route them correctly.




    When a service is successfully added in Xdesk 365, the status changes to "Completed", indicating that the task is finished.



    Add users for seamless task management and teamwork.




    When a user is successfully added in Xdesk 365, the status changes to "Completed", indicating that the task is finished.



    Set up an automated approval workflow for a smooth approval process.




    When an approval is successfully added in Xdesk 365, the status changes to "Completed", indicating that the task is finished.



    Create custom forms to improve collaboration and department-specific clarity.




    When an custom forms is successfully added in Xdesk 365, the status changes to "Completed", indicating that the task is finished.



    Configure general settings for customization and additional features.




    Once all steps are completed (100%), the "Proceed" button is enabled, allowing users to enter the Xdesk 365 application.



  14. Finally click on Publish button.


  15. Finally, refresh the page.

How to remove this solution available to all sites in the organization


  1. Now select the Xdesk 365, click on files and then the deploy icon.


  2. Now unselect the check box and finally click on deploy button.

There are two types of installations:

1. Installation via package upload in the app catalog site.
2. Installation from the SharePoint store (AppSource).

If you delete the package installation and reinstall the app from the SharePoint store, or vice versa, it will not impact or result in data loss. This is because the application's data is stored safely in SharePoint lists, which remain unaffected by the installation method.
